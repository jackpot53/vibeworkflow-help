# Git & GitHub 설치하기

<Screenshot src="https://encrypted-tbn0.gstatic.com/images?q=tbn:ANd9GcRvB4PVpG6ojDsNJGQIVW2q-2qAXc6M8ZEmkw&s" alt="Git & GitHub" />

구글 독스에서 문서를 편집하면 **"버전 기록"** 이 자동으로 남아요.\
언제, 누가, 무엇을 바꿨는지 확인하고 이전 상태로 되돌릴 수 있죠.

**Git은 코드 파일을 위한 버전 기록 시스템이에요.**\
코드 파일도 문서처럼 수시로 바뀌고, 여러 사람이 함께 작업하기 때문에 이런 이력 관리가 꼭 필요합니다.

<Callout type="info">
  **GitHub는 Git의 저장 공간이에요.** Git이 내 컴퓨터에서 이력을 기록하는 도구라면, GitHub는 그 이력을 인터넷 어딘가에 안전하게 보관해두는 저장소입니다. 구글 독스에서 문서가 구글 서버에 저장되는 것과 같은 개념이에요.
</Callout>

## GitHub 가입

사내 GitHub도 있지만, 이후 Vercel 배포를 위해 외부 GitHub 계정을 사용합니다.\
[**github.com**](https://github.com)에서 무료 개인 계정을 만드세요.\
가입 후 이메일 인증이 필요합니다.

<Callout type="info">
  자세한 가입 방법은 [GitHub에서 계정 생성하기](https://docs.github.com/ko/get-started/start-your-journey/creating-an-account-on-github) 공식 문서를 참고하세요.
</Callout>

## GitHub Desktop 다운로드 & 설치

macOS 12.0 이상에서 GitHub Desktop을 사용할 수 있습니다.

1. [GitHub Desktop 다운로드 페이지](https://github.com/apps/desktop?locale=ko-kr)로 이동합니다.
2. **Download for macOS**를 클릭합니다.
3. 다운로드 폴더에서 zip 파일을 더블 클릭해 압축을 풉니다.
4. GitHub Desktop 애플리케이션을 더블 클릭하면 실행됩니다.

![GitHub Desktop 설치 화면](/images/ctfassets-screenshot-windows-dark.png?w=2496&fm=webp&q=90)

설치 후 터미널에서 Git이 정상적으로 설치됐는지 확인합니다.

<ShellBlock>
```bash
git --version
```
</ShellBlock>

버전 숫자가 출력되면 완료입니다.

## GitHub와 Git 연결하기 (SSH 키 설정)

GitHub에 코드를 올리거나 가져올 때는 "이 사람이 진짜 계정 주인인가?"를 확인해야 합니다.\
이를 인증(Authentication)이라고 하며, SSH 키 방식을 사용합니다.

<TimelineTabs tabs={["SSH 키 생성", "공개키 확인", "GitHub에 SSH Key 등록", "연결 테스트"]}>
<div>

터미널에서 아래 명령어를 입력하세요.\
이메일은 GitHub 가입 시 사용한 이메일을 넣습니다.

<ShellBlock>
```bash
ssh-keygen -t ed25519 -C "내이메일@example.com"
```
</ShellBlock>

프롬프트가 나타나면 엔터를 몇 번 눌러 기본 경로에 저장합니다.

```output
~/.ssh/id_ed25519      # 개인키 (절대 공유하지 마세요)
~/.ssh/id_ed25519.pub  # 공개키 (GitHub에 등록할 키)
```

이메일이 기억나지 않으면 `git config user.email`로 확인할 수 있습니다.

</div>
<div>

아래 명령어로 공개키를 확인하고 전체 내용을 복사합니다.

<ShellBlock>
```bash
cat ~/.ssh/id_ed25519.pub
```
</ShellBlock>

출력된 `ssh-ed25519 AAAA...` 문자열 **전체**를 복사하세요.

</div>
<div>

![GitHub SSH Key 등록 화면](/images/ssh-and-gpg-keys.png)


1. GitHub 프로필 사진 클릭 -> **Settings** ->  **SSH and GPG keys** -> **New SSH key**
2. Title 입력 (예: `My MacBook`), **복사한 공개키** 붙여넣기
3. **Add SSH key** 클릭


</div>
<div>

<ShellBlock>
```bash
ssh -T git@github.com
```
</ShellBlock>

처음엔 호스트 신뢰 여부를 묻는 메시지가 나옵니다. `yes`를 입력하세요.\
아래 메시지가 나오면 인증 성공입니다.

```output
Hi username! You've successfully authenticated, but GitHub does not provide shell access.
```

</div>
</TimelineTabs>


## GitHub CLI 설치하기

GitHub 웹사이트에서 하는 작업들 — PR 만들기, 이슈 등록, 저장소 관리 — 을 **터미널 명령어로 바로 처리**할 수 있게 해주는 도구예요.

<Callout type="info">
  **CLI(Command Line Interface)** 란 마우스로 클릭하는 대신 명령어를 입력해서 프로그램을 조작하는 방식이에요. GitHub CLI는 GitHub 웹사이트를 터미널에서 쓸 수 있게 해주는 원격 제어기 같은 거예요.
</Callout>

클로드코드에게 "PR 만들어줘"라고 하면, 클로드코드가 내부적으로 GitHub CLI를 이용해 대신 처리해줍니다.\
GitHub CLI가 없으면 클로드코드의 PR 자동화 기능을 쓸 수 없어요.

<Callout type="tip">
  **PR(Pull Request)이란?** 내가 작업한 코드를 메인 코드에 합쳐달라는 요청이에요.\
  "이런 기능을 추가했는데 검토하고 반영해주세요"라고 GitHub에 알리는 방식이에요.
</Callout>

<PRFlowDiagram />

<Callout type="info">
  Homebrew가 설치되어 있어야 합니다.
  아직 설치하지 않으셨다면 [brew(Homebrew) 설치](./brew) 페이지를 먼저 진행하세요.

<ShellBlock>
```bash
brew install gh
```
</ShellBlock>

설치 후 GitHub 계정을 연결합니다.

<ShellBlock>
```bash
gh auth login --web --scopes repo,read:org,workflow
```
</ShellBlock>

프롬프트에 따라 진행하세요.

```output
? What account do you want to log into? GitHub.com
? What is your preferred protocol for Git operations on this host?
  > HTTPS                          ← HTTPS 선택
? Authenticate Git with your GitHub credentials? Yes  ← Yes 선택
! First copy your one-time code: ABCD-1234          ← 이 코드 복사
Press Enter to open github.com in your browser...   ← Enter 후 브라우저에서 코드 입력
```

```output
✓ Authentication complete.
✓ Configured git protocol
✓ Logged in as username
```

위 메시지가 나오면 모든 설정이 완료된 것입니다.
