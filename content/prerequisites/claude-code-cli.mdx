# 클로드코드 CLI 설치하기

<Screenshot src="/images/claude-code-title.png" alt="클로드코드 CLI" />

**CLI(Command Line Interface)** 란 마우스 클릭 대신 텍스트 명령어를 입력해서 프로그램을 실행하는 방식이에요.\
GUI(그래픽 인터페이스)가 Finder에서 폴더를 클릭하는 방식이라면, CLI는 터미널에 `cd my-project`처럼 직접 입력하는 방식이에요.

<span className="bg-yellow-200 dark:bg-yellow-800/50 rounded px-1 py-0.5 font-medium">클로드코드 CLI는 이 터미널에서 <code className="bg-yellow-300 dark:bg-yellow-700/60 px-1 rounded text-sm">claude</code> 명령어 하나로 실행하는 AI 개발 도구예요.</span>\
코드 읽기, 수정, 실행까지 자연어로 AI와 함께 할 수 있습니다.

<Steps>

### Node.js 설치

클로드코드 CLI를 설치하려면 **Node.js**와 **npm**이 필요합니다.

> **Node.js** — 자바스크립트를 실행하는 엔진 (실행 환경)\
> **npm** (Node Package Manager) — 필요한 라이브러리를 설치하고 관리하는 도구

<Callout type="info">
  Node.js를 설치하면 npm은 자동으로 함께 설치됩니다.
</Callout>

다음 링크에서 [Node.js](https://nodejs.org/ko/download)를 다운로드하여 설치합니다.

설치 후 터미널에서 아래 명령어로 정상 설치 여부를 확인합니다.

<ShellBlock>
```bash
node --version
npm --version
```
</ShellBlock>

```output
v22.13.1
10.9.2
```

버전 숫자가 출력되면 설치 완료입니다.

### 클로드코드 CLI 설치 및 실행

클로드코드 CLI는 로컬에 설치되어 터미널 기반으로 내 코드를 이해하고, 직접 수정·실행까지 해주는 AI 개발 도구입니다.
설치 방법 중 **Native Install(권장)** 방식으로 설치합니다.

터미널에 다음 명령어를 입력합니다.

<ShellBlock>
```bash
curl -fsSL https://claude.ai/install.sh | bash
```
</ShellBlock>

설치가 완료되면 아래와 같은 메시지가 출력됩니다.

```output
✔ Claude Code successfully installed!
  Version: 2.1.50
  Location: ~/.local/bin/claude
  Next: Run claude --help to get started

⚠ Setup notes:
  • Native installation exists but ~/.local/bin is not in your PATH. Run:

  echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc

✅ Installation complete!
```

**PATH 설정** : `claude` 명령어를 어느 위치의 터미널에서든 실행할 수 있도록 PATH를 등록합니다.

<Callout type="warning">
  PATH를 등록하지 않으면 터미널에서 `claude`를 입력했을 때 `command not found: claude` 오류가 발생해요.\
  설치는 됐지만 컴퓨터가 `claude`가 어디 있는지 몰라서 실행을 못 하는 상태입니다. 반드시 아래 명령어를 실행해주세요.
</Callout>

<ShellBlock>
```bash
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc
```
</ShellBlock>

**설치 확인** : 다음 명령어 입력 시 버전이 표시되면 설치가 완료된 것입니다.

<ShellBlock>
```bash
claude -v
```
</ShellBlock>

```output
2.1.87 (Claude Code)
```

### 첫 실행 및 초기 설정

작업할 프로젝트 폴더로 이동한 뒤 `claude` 명령어를 입력합니다.

<ShellBlock>
```bash
cd your-project   # 작업할 프로젝트 폴더로 이동
claude
```
</ShellBlock>

<Callout type="info">
  `your-project` 자리에 본인이 작업할 폴더 경로를 입력하세요.
</Callout>

### 터미널 상태 표시줄 설정

클로드코드를 사용하다 보면 **"지금 AI가 얼마나 대화를 기억하고 있지?"**, **"돈이 얼마나 쓰였지?"** 같은 게 궁금해져요.\
상태 표시줄은 터미널 하단에 항상 떠 있는 **미니 대시보드**예요. 지금 무슨 모델을 쓰는지, 대화 공간이 얼마나 찼는지를 한눈에 볼 수 있어요.

| 표시 항목 | 쉬운 설명 |
|---|---|
| **모델명** | 지금 어떤 Claude를 쓰고 있는지 (예: Opus 4.5, Sonnet 4.5) |
| **컨텍스트 사용량** | AI가 이 대화에서 기억할 수 있는 공간이 얼마나 찼는지. 꽉 차면 오래된 내용부터 잊어버려요 |
| **세션 비용** | 이번 대화에서 API 비용이 얼마나 쓰였는지 (구독 플랜은 해당 없음) |
| **현재 폴더** | 지금 터미널이 어느 폴더에 있는지 |
| **Git 브랜치** | 코드 버전 관리 중이라면 현재 작업 브랜치 이름 |
| **현재 시간** | 말 그대로 시계 |

<Screenshot src="/images/statusline-annotated.jpg" alt="터미널 상태 표시줄 예시 — 빨간 테두리 영역이 상태 표시줄" />

클로드코드 CLI 안에서 [`/statusline`](/reference/commands/statusline) 명령어에 원하는 내용을 자연어로 입력합니다.

<SlashBlock>
```
/statusline 모델 이름, 컨텍스트 진행 상태 바, 현재 디렉토리, Git 브랜치, 그리고 현재 시간을 표시해라
```
</SlashBlock>

클로드코드가 상태 표시줄 스크립트를 만들기 위해 명령어를 실행해도 되는지 물어봐요.

<ClaudeBlock>
```
Claude wants to execute a bash command:
  bash ~/.claude/statusline-command.sh

❯ 1. Yes          → 이번 한 번만 허용
  2. Yes, always  → 앞으로 이 명령어는 항상 자동으로 허용 (다시 안 물어봄)
  3. No           → 이번엔 실행하지 않음
```
</ClaudeBlock>

<Callout type="tip">
  **2번 Yes, always**를 선택하면 앞으로 같은 명령어가 나올 때 자동으로 허용돼요.\
  처음엔 1번으로 확인하고, 익숙해지면 2번을 선택해도 괜찮아요.

  앞으로 클로드코드로 작업하다 보면 이런 허가 요청 화면이 자주 등장해요. "클로드가 파일을 수정해도 될까요?", "이 명령어를 실행해도 될까요?" 같은 식으로요. 처음엔 낯설 수 있지만, 클로드가 내 컴퓨터에서 뭔가를 하기 전에 **항상 먼저 물어본다**는 뜻이에요. 익숙해질수록 흐름이 자연스럽게 느껴질 거예요.

</Callout>

<Callout type="info">
  **왜 매번 물어볼까요?** 클로드코드는 AI가 직접 내 컴퓨터에서 명령어를 실행하는 도구예요. 파일을 삭제하거나 코드를 바꾸는 것처럼 되돌리기 어려운 작업도 할 수 있거든요. 그래서 내가 원하지 않는 일이 몰래 일어나지 않도록, 중요한 행동 전에는 반드시 내 허락을 먼저 받도록 설계되어 있어요.
</Callout>

설정이 완료되면 다음과 같이 확인 메시지가 출력됩니다.

<ClaudeBlock>
```
⏺ Your statusline is configured! It will display:

  - Model name (cyan)
  - Context progress bar with percentage
  - Current directory (yellow, ~ for home)
  - Git branch in parentheses (yellow)
  - Current time HH:MM:SS (green)

  The configuration is saved in ~/.claude/settings.json
  with the script at ~/.claude/statusline-command.sh.
```
</ClaudeBlock>

<Callout type="info">
  설정 후 `/exit`으로 세션을 나갔다가 `claude`로 다시 입력하면 상태 표시줄이 활성화됩니다.
</Callout>

<Callout type="tip">
  스크립트 샘플이나 표시 항목 커스터마이징 방법은 [/statusline 가이드](/reference/commands/statusline)를 참고하세요.
</Callout>

</Steps>
