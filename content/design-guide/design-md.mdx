# DESIGN.md 만들기

## 디자인 시스템이란?

디자인 시스템은 제품 전체에서 일관된 UI를 만들기 위한 **규칙의 모음**이에요.

색상, 폰트, 간격, 버튼 스타일, 컴포넌트 사용 방식 — 이런 결정들을 한 곳에 정리해두면 어떤 페이지를 만들든 같은 느낌이 유지돼요. 디자이너와 개발자가 함께 쓰는 공통 언어라고 생각하면 돼요.

대표적인 디자인 시스템으로는 Google의 **Material Design**, Apple의 **Human Interface Guidelines**, Vercel의 **Geist** 등이 있어요.

---

## Claude Code에서 DESIGN.md가 왜 필요한가요?

Claude는 코드를 잘 짜지만, **취향은 없어요**.

아무런 기준 없이 "버튼 스타일을 바꿔줘"라고 하면 Claude는 그냥 무난한 선택을 해요. 매번 다른 색상, 들쭉날쭉한 간격, 일관성 없는 폰트 크기가 생길 수 있어요.

DESIGN.md는 이 문제를 해결해요. 프로젝트의 디자인 기준을 파일로 정리해두면, Claude가 새 컴포넌트를 만들거나 UI를 수정할 때 그 기준을 따라요.

| 없을 때 | 있을 때 |
|---------|---------|
| 버튼마다 색상이 달라요 | 항상 같은 primary 색상을 써요 |
| 간격이 제각각이에요 | 8px 단위 간격 규칙을 지켜요 |
| 폰트 크기가 무작위예요 | 정해진 타입 스케일을 따라요 |
| 컴포넌트 스타일이 섞여요 | shadcn/ui 규칙 안에서 움직여요 |

<Callout type="tip">
CLAUDE.md가 코딩 규칙을 알려준다면, DESIGN.md는 디자인 규칙을 알려줘요. 둘을 함께 쓰면 Claude가 훨씬 일관된 결과물을 만들어요.
</Callout>

## DESIGN.md 핵심 구성요소

DESIGN.md는 크게 5가지 항목으로 구성해요.

### 1. 색상 (Colors)

브랜드 색상과 역할을 정의해요. 색상 이름보다 **용도 중심**으로 쓰는 게 중요해요.

```md
## Colors
- Primary: #6366F1 — 주요 버튼, 링크, 강조 요소
- Background: #FFFFFF / #0F0F0F — 라이트/다크 배경
- Muted: #F4F4F5 — 보조 배경, 비활성 영역
- Destructive: #EF4444 — 삭제, 오류, 경고
```

### 2. 타이포그래피 (Typography)

폰트 종류와 크기 체계를 정해요. Claude가 텍스트 요소를 만들 때 이 규칙을 따라요.

```md
## Typography
- Font: Pretendard (sans-serif)
- 제목(h1): 36px / font-bold
- 소제목(h2): 24px / font-semibold
- 본문: 16px / font-normal
- 보조 텍스트: 14px / text-muted-foreground
```

### 3. 간격 (Spacing)

레이아웃의 리듬을 만드는 간격 기준이에요. 8px 단위 시스템이 가장 많이 쓰여요.

```md
## Spacing
- 기본 단위: 8px (Tailwind 기준 2)
- 컴포넌트 내부 패딩: 16px (4)
- 섹션 간 간격: 64px (16)
- 카드 간 gap: 24px (6)
```

### 4. 컴포넌트 스타일 (Components)

자주 쓰는 컴포넌트의 기본 스타일 규칙을 적어요.

```md
## Components
- Button: rounded-lg, size default, primary variant 기본
- Card: rounded-xl, border border-border, shadow-sm
- Input: rounded-md, h-10, focus:ring-primary
- Badge: rounded-full, text-xs
```

### 5. 톤 & 무드 (Tone & Mood)

전체적인 디자인 방향성을 한두 문장으로 정의해요. Claude가 새 UI를 제안할 때 이 방향을 기준으로 삼아요.

```md
## Tone & Mood
미니멀하고 깔끔한 SaaS 스타일. 불필요한 장식 없이 콘텐츠 중심.
다크 모드 우선. 여백을 충분히 활용해서 호흡감을 줘요.
```

<Callout type="tip">
처음부터 완벽하게 채울 필요 없어요. 색상과 폰트만 있어도 Claude의 결과물이 달라져요. 프로젝트를 진행하면서 조금씩 채워가면 돼요.
</Callout>

  <Tabs.Tab>
    ### 구글 스티치로 만들기

    [Google Stitch](https://stitch.withgoogle.com)는 Google Labs의 AI UI 디자인 도구예요.\
    Gemini 기반으로 URL이나 이미지를 입력하면 UI를 생성하고 HTML/CSS 코드까지 바로 뽑아줘요.\
    이 코드를 Claude에게 주면 DESIGN.md를 만들 수 있어요.

    **방법 1 — URL로 디자인 토큰 추출**

    1. [stitch.withgoogle.com](https://stitch.withgoogle.com)에 접속해요
    2. 입력창에 참고할 사이트 URL을 붙여넣어요
    3. Stitch가 해당 사이트의 색상, 폰트 등 디자인 토큰을 추출하고 UI를 생성해요
    4. 생성된 HTML/CSS 코드를 복사해서 Claude에게 전달해요

    <VibeBlock>
    ```
    아래는 Google Stitch로 추출한 디자인 코드야.
    이걸 분석해서 우리 프로젝트의 DESIGN.md를 만들어줘.

    [Stitch에서 복사한 HTML/CSS 코드 붙여넣기]
    ```
    </VibeBlock>

    **방법 2 — 이미지 첨부로 UI 생성**

    1. [stitch.withgoogle.com](https://stitch.withgoogle.com)에 접속해요
    2. 입력창에 디자인 시안이나 스크린샷 이미지를 첨부해요
    3. Stitch가 이미지를 분석해서 UI를 생성해요
    4. 코드 아이콘을 클릭해서 **HTML/Tailwind CSS 코드를 다운로드**해요
    5. 다운로드한 코드를 Claude에게 전달해서 DESIGN.md를 만들어요

    <VibeBlock>

    </VibeBlock>

      Stitch는 DESIGN.md를 직접 내보내지 않아요. HTML/Tailwind CSS 코드를 다운로드한 뒤 Claude에게 전달하면 Claude가 DESIGN.md로 변환해줘요. URL 입력과 이미지 첨부는 동시에 사용할 수 없으니 하나만 선택하세요.
    </Callout>

    ### 피그마로 만들기

    피그마 파일 URL을 Claude에게 주면 디자인을 직접 분석해서 DESIGN.md를 만들어줘요.

    **순서**

    1. 피그마에서 공유하고 싶은 파일이나 페이지를 열어요
    2. 우측 상단 **Share** 버튼을 눌러 링크를 복사해요 (Anyone with the link로 설정)
    3. Claude에게 URL을 전달해요

    <VibeBlock>
    ```
    https://www.figma.com/file/xxxx/... 이 피그마 파일을 분석해서
    DESIGN.md를 만들어줘.
    색상 팔레트, 타이포그래피, 컴포넌트 스타일 위주로 정리해줘.
    ```
    </VibeBlock>

    <Callout type="tip">
      디자인 시스템 페이지나 컴포넌트 페이지 URL을 주면 더 정확하게 추출해요.
    </Callout>
  </Tabs.Tab>

  <Tabs.Tab>
    ### 웹사이트 URL로 만들기

    참고하고 싶은 디자인의 웹사이트 URL을 Claude에게 주면, Claude가 직접 분석해서 DESIGN.md를 만들어줘요.

    **순서**

    1. 디자인 레퍼런스로 삼고 싶은 사이트를 찾아요
    2. URL을 복사해서 Claude에게 전달해요

    <VibeBlock>
    ```
    https://linear.app 이 사이트의 디자인 스타일을 분석해서
    우리 프로젝트용 DESIGN.md로 만들어줘.
    색상, 타이포그래피, 컴포넌트 스타일 위주로.
    ```
    </VibeBlock>

    <Callout type="tip">
      "이 사이트처럼 만들어줘"보다 "이 사이트의 디자인 시스템을 DESIGN.md로 정리해줘"가 더 구체적인 결과를 줘요.
    </Callout>
  </Tabs.Tab>

  <Tabs.Tab>
    ### 이미지 첨부로 만들기

    디자인 시안, 스크린샷, 무드보드 이미지를 Claude에게 첨부하면 시각적으로 분석해서 DESIGN.md를 만들어줘요.

    **순서**

    1. 참고할 디자인 이미지를 준비해요 (시안, 스크린샷, 레퍼런스 무드보드 등)
    2. Claude Code 채팅창에 이미지를 드래그하거나 붙여넣어요
    3. 아래처럼 요청해요

    <VibeBlock>
    ```
    첨부한 디자인 이미지를 분석해서 DESIGN.md를 만들어줘.
    색상 팔레트, 폰트 느낌, 컴포넌트 스타일, 전체적인 톤 & 무드를 정리해줘.
    ```
    </VibeBlock>

    <Callout type="tip">
      여러 장을 한 번에 첨부해도 돼요. "이 이미지들의 공통된 디자인 언어를 추출해줘"라고 하면 일관된 스펙을 뽑아줘요.
    </Callout>
  </Tabs.Tab>

  <Tabs.Tab>
    ### 기존 코드 기반으로 만들기

    이미 구현된 코드가 있다면 Claude가 코드를 읽고 현재 디자인 패턴을 역으로 추출해줘요.\
    새로 시작하는 게 아니라 기존 프로젝트를 정리할 때 유용해요.

    **순서**

    1. 프로젝트 루트에서 Claude Code를 열어요
    2. 아래처럼 요청해요

    <VibeBlock>
    ```
    현재 프로젝트의 컴포넌트와 스타일 코드를 분석해서
    DESIGN.md를 만들어줘.
    실제로 사용 중인 색상, 폰트, 간격, 컴포넌트 패턴을 기준으로 정리해줘.
    ```
    </VibeBlock>

    <Callout type="tip">
      코드 기반으로 만든 DESIGN.md는 현재 상태의 스냅샷이에요. 만든 후에 앞으로의 방향도 함께 추가하면 더 유용해요.
    </Callout>
  </Tabs.Tab>
