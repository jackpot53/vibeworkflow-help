# 명령어 만들기

Claude Code에서 `/`로 시작하는 명령어는 내장 명령어 외에도 **직접 만들 수 있습니다.**
자주 쓰는 작업 절차, 프로젝트별 규칙, 반복 작업 등을 명령어로 등록해두면 매번 긴 설명을 입력하지 않아도 돼요.

이렇게 직접 만든 명령어를 **명령어(COMMAND)** 라고 합니다.

---

## 어떻게 만드나요?

명령어는 **마크다운 파일(.md)** 하나로 만들어요.
파일 맨 위에 YAML 형식으로 설정을 적고, 그 아래에 Claude에게 전달할 지시문을 작성합니다.

```markdown
---
name: my-command
description: 자주 쓰는 작업을 자동화합니다
---

여기에 Claude에게 줄 지시문을 작성하세요.
예: 현재 브랜치의 변경 사항을 요약하고 PR 설명을 초안해줘.
```

이 파일을 다음 위치 중 하나에 저장하면 바로 `/my-command`로 사용할 수 있어요.

| 저장 위치 | 적용 범위 |
|-----------|-----------|
| `~/.claude/commands/` | 내 컴퓨터의 모든 프로젝트에서 사용 |
| `.claude/commands/` | 현재 프로젝트에서만 사용 |

<Callout type="tip">
  명령어 파일 이름이 명령어 이름이 됩니다.\
  `pr-review.md`로 저장하면 `/pr-review`로 호출할 수 있어요.
</Callout>

---

## 설정 필드 (frontmatter)

파일 맨 위 `---` 사이에 들어가는 설정 항목들이에요. 모두 선택 사항이지만 `description`은 넣는 게 좋아요.

| 필드 | 설명 |
|------|------|
| `name` | 슬래시 명령어 이름. 생략하면 파일명이 사용됨 |
| `description` | 명령어가 하는 일. `/` 자동완성 목록과 Claude 검색에 표시됨 |
| `argument-hint` | 자동완성 중 표시되는 인수 힌트 (예: `[issue-number]`, `[filename]`) |
| `allowed-tools` | 이 명령어 실행 시 권한 확인 없이 허용할 도구 목록 |
| `model` | 이 명령어 실행 시 사용할 모델 (`haiku`, `sonnet`, `opus`) |
| `effort` | 모델 노력 수준 (`low`, `medium`, `high`, `max`) |
| `context` | `fork`로 설정하면 독립된 서브에이전트 컨텍스트에서 실행 |
| `agent` | `context: fork` 사용 시 서브에이전트 유형 (기본값: `general-purpose`) |
| `paths` | 명령어가 활성화될 파일 경로 패턴 (Glob). 설정하면 해당 파일 작업 시에만 자동 로드 |
| `disable-model-invocation` | `true`로 설정하면 Claude가 이 명령어를 자동 호출하지 못함 |
| `user-invocable` | `false`로 설정하면 `/` 메뉴에서 숨겨짐. 백그라운드 지식으로만 활용 |
| `shell` | `` !`command` `` 블록에서 사용할 셸 (`bash` 기본값, `powershell` 가능) |
| `hooks` | 이 명령어에만 적용할 라이프사이클 훅 |

## 예제

아래 파일들을 `.claude/commands/` 아래 저장하면 바로 슬래시 명령어로 사용할 수 있습니다.

---

### commit.md — 커밋 메시지 생성기

staged된 변경 사항을 분석해 한국어 커밋 메시지를 작성합니다.

```markdown
---
name: commit
description: 변경 사항을 분석해 커밋 메시지를 작성합니다
allowed-tools: Bash(git diff), Bash(git status)
---

현재 staged된 변경 사항을 분석해서 명확하고 간결한 커밋 메시지를 작성해줘.
형식: <타입>: <요약> (한국어로)
```

<SlashBlock>
```
> /commit
```
</SlashBlock>

---

### pr-draft.md — PR 설명 초안

비교할 브랜치를 인수로 넘기면 diff를 분석해 PR 설명을 초안합니다.

```markdown
---
name: pr-draft
description: 현재 브랜치 기준으로 PR 설명을 초안합니다
argument-hint: [base-branch]
model: sonnet
---

현재 브랜치와 `$ARGUMENTS` 브랜치의 diff를 분석해서 PR 설명을 작성해줘.
- 변경 사항 요약
- 테스트 방법
- 주의 사항
```

<SlashBlock>
```
> /pr-draft main
```
</SlashBlock>

---

### translate.md — 번역 명령어

단순 번역처럼 가벼운 작업은 Haiku로 지정해 응답 속도를 높입니다.

```markdown
---
name: translate
description: 선택한 텍스트를 한국어로 번역합니다
model: haiku
effort: low
---

다음 텍스트를 자연스러운 한국어로 번역해줘: $ARGUMENTS
```

<SlashBlock>
```
> /translate Hello, world!
```
</SlashBlock>

<Callout type="info">
  `$ARGUMENTS`는 명령어 실행 시 사용자가 입력한 인수로 대체됩니다.\
  예: `/translate Hello world` → `$ARGUMENTS`가 `Hello world`로 치환됩니다.
</Callout>
