# 베스트 프랙티스

Claude Code 초기 개발자 Boris Cherny가 직접 공유한 실전 팁이에요.

## 01. 서브에이전트 대신, 워크플로우엔 커맨드를 써라

<Callout type="tip">
  [Boris Tip](https://x.com/bcherny/status/2007179847949500714) — Boris Cherny (Claude Code 초기 개발자)
</Callout>

> 저는 하루에 여러 번 반복하는 모든 "내부 루프" 워크플로우에 슬래시 커맨드를 사용합니다.
> 이렇게 하면 반복적인 프롬프팅을 줄일 수 있고, Claude도 이 워크플로우를 활용할 수 있습니다.
> 커맨드는 git에 커밋되며 `.claude/commands/`에 저장됩니다.

**"내부 루프"가 뭔가요?**

개발하면서 하루에도 수십 번 반복하는 짧은 작업 사이클이에요. 예를 들면 코드 수정 → 테스트 → 커밋 → 푸시 같은 흐름이에요. 이걸 매번 말로 설명하지 않고 `/명령어` 하나로 끝내는 거예요.

Boris와 Claude가 매일 수십 번씩 쓰는 `/commit-push-pr`이 대표적인 예예요.

<SlashBlock>
```
> /commit-push-pr
```
</SlashBlock>

이 커맨드 하나로 아래 세 작업이 자동으로 처리돼요.

- **commit** — 변경된 파일을 git에 저장
- **push** — 원격 저장소에 업로드
- **PR 생성** — Pull Request(코드 리뷰 요청) 자동 오픈

매번 "커밋하고, 푸시하고, PR 열어줘"라고 말하는 대신 한 번에 끝나요. 실제 구현은 [GitHub에서 확인](https://github.com/anthropics/claude-code/blob/main/.claude/commands/commit-push-pr.md)할 수 있어요.

## 02. 하루에 여러 번 반복하는 작업은 전부 커맨드로 만들어라

하루에 두 번 이상 같은 말을 Claude에게 하고 있다면, 그건 커맨드로 만들 신호예요.

<Callout type="info">
  커맨드로 만들어두면 나뿐 아니라 팀원 모두가 같은 방식으로 쓸 수 있어요. `.claude/commands/` 폴더에 저장하고 git에 커밋하면 팀 전체가 공유해요.
</Callout>

## 03. 하루에 두 번 이상 하는 건 스킬이나 커맨드로 만들어라

<Callout type="tip">
  [Boris Tip](https://x.com/bcherny/status/2017742748984742078) — Boris Cherny (Claude Code 초기 개발자)
</Callout>

> 나만의 스킬을 만들고 git에 커밋하세요. 모든 프로젝트에서 재사용할 수 있습니다.

실제로 팀에서 활용하는 예시들이에요.

**`/techdebt` 커맨드**
매 세션 끝날 때 실행해서 중복 코드를 찾아 정리해요. "기술 부채"가 쌓이지 않게 습관적으로 쓸 수 있어요.

**도메인 특화 에이전트 커맨드**
예를 들어 데이터 분석가라면, dbt 모델 작성 → 코드 리뷰 → 개발 환경 테스트를 한 번에 처리하는 커맨드를 만들어 쓸 수 있어요.

<Callout type="info">
  핵심은 **"두 번 이상 반복하면 자동화한다"**는 원칙이에요. 처음엔 번거로워 보여도, 한 번 만들어두면 매일 시간을 아낄 수 있어요.
</Callout>
