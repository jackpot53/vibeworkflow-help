# 명령어란?

명령어는 Claude Code에서 <mark>내가 원할 때 직접 실행하는 슬래시 명령어</mark>예요. `/명령어`를 입력하면 미리 정의된 작업을 바로 시작해요.

---

## 왜 명령어가 필요한가요?

"커밋하고 PR 만들어줘", "코드 리뷰해줘" 같은 작업을 매번 자연어로 설명하는 건 번거로워요. 명령어로 만들어두면 한 번의 입력으로 복잡한 워크플로우를 실행할 수 있어요.

- 반복 작업을 한 줄로 실행할 수 있어요
- 팀 전체가 동일한 방식으로 작업할 수 있어요
- 내가 원하는 타이밍에 정확히 실행할 수 있어요

---

## Command vs Skill

명령어와 스킬은 비슷해 보이지만 역할이 달라요.

| | Command | Skill |
| :--- | :--- | :--- |
| **호출 방식** | 내가 `/명령어`로 직접 실행 | Claude가 상황 판단해서 자동 로드 |
| **용도** | 내가 실행하고 싶을 때 | Claude가 알아서 써야 할 때 |

예를 들어 `/commit-push-pr`이라는 커밋 → 푸시 → PR 생성 워크플로우가 있다고 해볼게요.

**Command로 만든 경우 (올바른 방법)**

<VibeBlock>
```
코드 수정 다 했어. /commit-push-pr
```
</VibeBlock>

<ClaudeBlock>
```
커밋 메시지 작성 중...
git commit, push, PR 생성 완료했어요.
```
</ClaudeBlock>

내가 "다 했다"고 판단한 시점에 직접 실행했어요.

**Skill로 만든 경우 (위험한 이유)**

Skill은 Claude가 대화 흐름을 보고 자동으로 불러올 수 있어요. 예를 들어 파일 수정 작업이 끝났다고 판단하는 순간 Claude가 알아서 커밋·푸시·PR을 실행해버릴 수 있어요. 내가 아직 검토도 안 했는데요.

<mark>실행 타이밍을 내가 결정해야 하는 작업은 Command, Claude가 알아서 적용해도 되는 지식·능력은 Skill</mark>로 만드는 게 기준이에요.

---

## 명령어 종류

**내장 명령어** — Claude Code에 기본으로 탑재된 명령어예요.

<SlashBlock>
```
/plan
/memory
/compact
```
</SlashBlock>

**커스텀 명령어** — 내가 직접 만든 명령어예요. `.claude/commands/` 폴더에 마크다운 파일로 저장해요.

<SlashBlock>
```
/commit-push-pr
/code-review
```
</SlashBlock>
