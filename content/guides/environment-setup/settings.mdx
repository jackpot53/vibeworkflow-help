# Claude Code 설정

Claude Code는 사용자의 필요에 맞게 동작을 구성할 수 있는 다양한 설정을 제공합니다.

대화형 세션에서 `/config` 명령을 실행하면 탭 형식의 설정 인터페이스가 열려 상태 정보를 보고 구성 옵션을 수정할 수 있습니다.

<SlashBlock>
```
/config
```
</SlashBlock>

```output
  Configure Claude Code preferences

  ╭────────────────────────────────────────────────────────────────╮
  │ ⌕ Search settings...                                           │

    Auto-compact                              true

    Respect .gitignore in file picker         true
    Always copy full response (skip /copy     false
  picker)
    Auto-update channel                       latest
    Theme                                     Dark mode
    Notifications                             Auto
    Output style                              default

  ❯ Model                                     Default (recommended)
    Auto-connect to IDE (external terminal)   false

```

<Callout type="tip">
  `/config`는 자주 쓰는 설정을 빠르게 바꾸는 용도입니다. 팀 공유나 세밀한 제어가 필요한 설정은 `settings.json` 파일을 직접 편집하세요.
</Callout>

## 권한 설정

Claude Code가 어떤 작업을 **자동으로 해도 되는지**, **하기 전에 먼저 물어봐야 하는지**, **절대 하면 안 되는지**를 지정하는 설정입니다.

`settings.json` 파일의 `"permissions"` 항목 아래에 작성합니다.

<FileBlock filename="settings.json">
```json
{
  "permissions": {
    "allow": ["Bash(git diff *)"],
    "deny": ["Read(./.env)"]
  }
}
```
</FileBlock>

| 키 | 설명 | 예시 |
|:---|:-----|:-----|
| `allow` | 확인 없이 **자동으로 허용**할 작업 목록입니다. 자주 쓰는 안전한 명령을 등록해 두면 매번 승인하지 않아도 됩니다. | `[ "Bash(git diff *)" ]` |
| `ask` | 실행 전에 **한 번 더 확인**을 요청할 작업 목록입니다. 중요한 작업을 실수로 그냥 넘기지 않도록 제동을 걸 때 씁니다. | `[ "Bash(git push *)" ]` |
| `deny` | **절대 허용하지 않을** 작업 목록입니다. 비밀번호 파일이나 민감한 폴더에 대한 접근을 원천 차단할 때 씁니다. | `[ "WebFetch", "Bash(curl *)", "Read(./.env)", "Read(./secrets/**)" ]` |
| `additionalDirectories` | Claude가 접근할 수 있는 **추가 폴더** 목록입니다. 기본적으로 현재 프로젝트 폴더만 열람 가능하며, 다른 폴더도 필요할 때 여기에 추가합니다. | `[ "../docs/" ]` |
| `defaultMode` | Claude Code를 열 때 적용되는 **기본 권한 수준**입니다. | `"acceptEdits"` |
| `disableBypassPermissionsMode` | `"disable"`로 설정하면 모든 권한 확인을 건너뛰는 위험 모드가 활성화되지 않도록 막습니다. 팀 환경에서 실수 방지용으로 유용합니다. | `"disable"` |

### 권한 규칙 구문

권한 규칙은 `도구명` 또는 `도구명(조건)` 형태로 작성합니다. 조건이 없으면 해당 도구의 **모든 사용**에 적용되고, 조건을 붙이면 **특정 상황에만** 적용됩니다.

규칙은 항상 **거부 → 확인 요청 → 허용** 순서로 평가되며, 가장 먼저 일치하는 규칙이 적용됩니다.

| 규칙 | 적용 범위 |
|:-----|:---------|
| `Bash` | 모든 터미널 명령 |
| `Bash(npm run *)` | `npm run`으로 시작하는 명령만 |
| `Read(./.env)` | 프로젝트 루트의 `.env` 파일 읽기만 |
| `WebFetch(domain:example.com)` | `example.com` 주소로의 요청만 |

<Callout type="tip">
  `*`는 와일드카드로, "그 뒤에 무엇이든"을 의미합니다. `Bash(git *)` 라고 쓰면 `git`으로 시작하는 모든 명령(`git status`, `git commit` 등)에 해당합니다.
</Callout>

<Callout type="info">
  와일드카드 동작 세부사항, `Read` · `Edit` · `WebFetch` · `MCP` · `Agent` 도구별 패턴, Bash 패턴의 보안 제한에 대한 완전한 규칙 구문은 [권한 규칙 구문](/ko/permissions#permission-rule-syntax)을 참고하세요.
</Callout>
