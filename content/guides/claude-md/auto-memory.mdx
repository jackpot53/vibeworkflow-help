# 자동 메모리 (Auto Memory)

## 자동 메모리란?

보통 Claude는 세션이 끝나면 모든 걸 잊어요. 다음 날 다시 열면 백지 상태예요.

자동 메모리를 켜두면 달라져요. <mark>Claude가 작업하면서 스스로 노트를 저장</mark>하고, 다음 세션에 그 내용을 이미 알고 있는 상태로 시작해요. 여러분이 아무것도 작성하지 않아도요.

Claude가 스스로 저장하는 내용은 이런 것들이에요:

- 빌드 명령어 (`npm run dev`가 아니라 `pnpm dev` 써야 한다 등)
- 디버깅에서 발견한 패턴
- 아키텍처 결정 사항
- 코드 스타일 선호도와 워크플로우 습관

실제로 어떤 순간에 저장되는지 시뮬레이션으로 볼게요.

<AutoMemorySimDemo />

바로 이 순간이에요. Claude가 교정을 받고 "이건 다음에도 필요한 정보다"라고 판단해서 자동으로 저장했어요. 이때 `MEMORY.md`에 이런 내용이 추가돼요.

<FileBlock filename="~/.claude/projects/my-project/memory/MEMORY.md">
```markdown
## 커밋 규칙
- 커밋 메시지는 한국어로 작성
- 메시지 앞에 관련 이모지 붙이기 (예: 🐛 버그 수정, ✨ 기능 추가)
```
</FileBlock>

다음 세션에서 Claude는 MEMORY.md를 먼저 확인하고, 따로 말하지 않아도 처음부터 규칙을 지켜요.

<Callout type="info">
  Claude는 매 세션마다 무조건 저장하지 않아요. <mark>이 내용이 다음에도 유용할까?</mark>를 스스로 판단한 뒤 저장해요.
</Callout>

---

## CLAUDE.md vs MEMORY.md — 뭐가 다른가요?

자동 메모리는 `MEMORY.md`라는 새 파일을 만들어요. CLAUDE.md와 역할이 달라요.

| | CLAUDE.md | MEMORY.md |
| :--- | :--- | :--- |
| **누가 작성?** | 나(개발자) | Claude 자신 |
| **내용** | 프로젝트 규칙, 명령어, 컨벤션 | 작업 중 배운 것들, 교정된 내용 |
| **예시** | "TypeScript strict mode 사용" | "pdflatex 말고 XeLaTeX 써야 함" |
| **관리** | 직접 편집 | `/memory`로 확인·편집·삭제 |

한 줄로 정리하면 이래요.

> **CLAUDE.md**는 내가 Claude에게 내리는 **지시서**,
> **MEMORY.md**는 Claude가 스스로 적는 **학습 노트**

둘이 충돌하면 <mark>CLAUDE.md가 우선</mark>해요.

## 실제 워크플로우

<StepBadge step={1} /> **세션 시작** — Claude가 CLAUDE.md와 MEMORY.md를 둘 다 읽고 시작해요.

<StepBadge step={2} /> **작업 중 교정** — 내가 "console.log 말고 logger 써줘"라고 하면 Claude가 판단해서 MEMORY.md에 저장해요.

<StepBadge step={3} /> **다음 세션** — Claude가 그 내용을 이미 알고 있는 상태로 시작해요. 다시 말 안 해도 돼요.

<StepBadge step={4} /> **확인·편집** — `/memory`를 실행하면 Claude가 저장한 내용을 전부 볼 수 있고, 일반 마크다운 파일이라 수정하거나 삭제할 수 있어요.

## 저장 위치

각 프로젝트의 자동 메모리는 내 컴퓨터의 아래 경로에 저장돼요.

<FileTree root={{
  name: "~/.claude/projects/<project>/memory/",
  children: [
    { name: "MEMORY.md", note: "인덱스 파일 — 모든 세션에 로드됨" },
    { name: "debugging.md", note: "디버깅 패턴 상세 노트" },
    { name: "api-conventions.md", note: "API 설계 결정" },
  ]
}} />

`MEMORY.md`는 인덱스 역할을 해요. Claude는 세션 전반에 걸쳐 이 파일들을 읽고 쓰면서 내용을 관리해요.

<Callout type="info">
  자동 메모리는 **내 컴퓨터에만 저장**돼요. 같은 git 저장소의 worktree끼리는 공유되지만, 다른 컴퓨터나 클라우드 환경과는 공유되지 않아요.
</Callout>

## 작동 방식

`MEMORY.md`의 처음 <mark>200줄이 모든 세션 시작 시 자동으로 로드</mark>돼요. 200줄을 넘는 내용은 시작 시 로드되지 않아요.

Claude는 내용이 길어지면 `debugging.md`, `patterns.md` 같은 주제별 파일로 분리해서 `MEMORY.md`를 간결하게 유지해요. 주제 파일들은 시작 시 로드되지 않고, Claude가 필요할 때 직접 읽어요.

Claude Code 화면에 **"메모리 작성"** 또는 **"메모리 회상"** 이라고 표시되면, Claude가 메모리 파일을 읽거나 쓰고 있는 거예요.

## 활성화 및 비활성화

자동 메모리는 기본적으로 켜져 있어요.

**토글 방법 1 — `/memory` 명령어**

<SlashBlock>
```
> /memory
```
</SlashBlock>

자동 메모리 토글을 눌러 켜거나 끌 수 있어요.

**토글 방법 2 — 설정 파일**

<FileBlock filename="~/.claude/settings.json">
```json
{
  "autoMemoryEnabled": false
}
```
</FileBlock>

**토글 방법 3 — 환경 변수**

<ShellBlock>
```bash
CLAUDE_CODE_DISABLE_AUTO_MEMORY=1
```
</ShellBlock>

## `/memory`로 확인하기

`/memory` 명령을 실행하면 현재 세션에 로드된 모든 CLAUDE.md와 자동 메모리 파일 목록을 볼 수 있어요.

<SlashBlock>
```
> /memory
```
</SlashBlock>

<ClaudeBlock>
```
  Memory
    Auto-memory: on
    Auto-dream: off · never
  ❯ 1. Project memory   Checked in at ./CLAUDE.md
    2. User memory      Saved in ~/.claude/CLAUDE.md
    3. Open auto-memory folder
```
</ClaudeBlock>

파일을 선택하면 편집기에서 바로 열려요. 불필요한 내용은 직접 삭제할 수 있어요.

Claude에게 직접 기억을 요청할 수도 있어요.

<VibeBlock>
```
항상 npm 말고 pnpm을 사용한다는 걸 기억해줘
```
</VibeBlock>

<VibeBlock>
```
API 테스트할 때 로컬 Redis 인스턴스가 필요하다는 걸 기억해줘
```
</VibeBlock>

CLAUDE.md에 추가하고 싶은 내용은 이렇게 요청하면 돼요.

<VibeBlock>
```
이걸 CLAUDE.md에 추가해줘
```
</VibeBlock>

## 문제 해결

### Claude가 CLAUDE.md를 따르지 않아요

CLAUDE.md는 강제 명령이 아니라 참고 자료예요. Claude가 읽고 따르려 하지만, 지침이 모호하거나 서로 충돌하면 놓칠 수 있어요.

확인해볼 것들:

- `/memory`를 실행해서 CLAUDE.md 파일이 목록에 보이는지 확인하세요. 안 보이면 Claude가 파일을 찾지 못하는 거예요.
- 지침을 더 구체적으로 작성해보세요. "코드를 제대로 포맷합니다" 보다 "2칸 들여쓰기 사용"이 훨씬 잘 작동해요.
- 여러 CLAUDE.md 파일에서 서로 충돌하는 지침이 없는지 확인하세요.

### 자동 메모리가 뭘 저장했는지 모르겠어요

<SlashBlock>
```
> /memory
```
</SlashBlock>

실행 후 **자동 메모리 폴더 열기**를 선택하면 Claude가 저장한 모든 내용을 볼 수 있어요. 전부 일반 마크다운 파일이라 직접 읽고, 수정하고, 삭제할 수 있어요.
