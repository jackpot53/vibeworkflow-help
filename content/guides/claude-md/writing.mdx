# CLAUDE.md 작성법

## 이런 상상을 해봅시다

당신에게 아주 유능한 팀원이 생겼습니다.
모르는 게 없고, 시키는 건 뭐든 빠르게 해결해줘요.

그런데 한 가지 문제가 있습니다. **이 팀원은 매일 아침, 어제의 기억을 모두 잃고 출근합니다.**

첫날, 당신은 꼼꼼하게 설명했어요.

> "우리 서비스 이름은 ○○이고, 고객은 이런 분들이고, 문서는 이 형식으로 써야 해."

팀원은 완벽하게 이해하고 일을 해냈습니다.

다음 날 아침, 팀원이 다시 왔습니다.

> "안녕하세요! 저 오늘 처음 왔는데요, 뭘 도와드릴까요?"

어제 했던 설명을 또 해야 합니다. 그다음 날도, 그다음 날도.

**CLAUDE.md는 이 문제를 해결합니다.**
팀원 책상 위에 놓인 업무 안내서처럼, Claude가 매일 가장 먼저 읽는 파일입니다.
프로젝트의 규칙, 작업 방식, 중요한 맥락을 한 번만 적어두면
Claude는 매 세션마다 그 내용을 바탕으로 대화를 시작합니다.

---

## Claude가 프로젝트를 기억하는 방법

Claude Code 세션에 처음 진입할 때마다 새로운 컨텍스트 윈도우로 시작됩니다. 즉, Claude는 이전 대화 내용을 기억하지 못한 채 매번 새로 시작합니다.<br/>
**CLAUDE.md 파일**은 세션이 시작될 때마다 자동으로 Claude에 전달되어 지속적인 지침을 제공합니다.

---

## CLAUDE.md란

CLAUDE.md 파일은 프로젝트, 개인 워크플로우 또는 전체 조직에 대해 Claude에 지속적인 지침을 제공하는 마크다운 파일입니다.
이러한 파일을 일반 텍스트로 작성하면 Claude가 모든 세션의 시작 시 이를 읽습니다.

Claude는 이 내용을 절대적인 명령이 아닌 참고 자료로 활용합니다 — 지침이 모호하거나 서로 충돌하면 Claude가 스스로 판단해 처리할 수 있어요.<br/>
지침이 더 구체적이고 간결할수록 Claude가 더 일관되게 따릅니다.

| | CLAUDE.md 파일 |
| :--- | :--- |
| **작성자** | 사용자 |
| **포함 내용** | 지침 및 규칙 |
| **범위** | 프로젝트, 사용자 또는 조직 |
| **로드 대상** | 모든 세션 |
| **사용 목적** | 코딩 표준, 워크플로우, 프로젝트 아키텍처 |

Claude의 동작을 안내하려면 CLAUDE.md 파일을 사용합니다.

---

## CLAUDE.md 파일을 어디에 배치할지 선택

CLAUDE.md 파일은 여러 위치에 있을 수 있으며, 각 위치는 다른 범위를 가집니다.<br/>
더 구체적인 위치가 더 광범위한 위치보다 우선합니다.

| 범위 | 위치 | 공유 대상 |
| :--- | :--- | :--- |
| **프로젝트 지침** | `./CLAUDE.md` 또는 `./.claude/CLAUDE.md` | 소스 제어를 통한 팀 멤버 |
| **사용자 지침** | `~/.claude/CLAUDE.md` | 본인만(모든 프로젝트) |
| **로컬 지침** | `./CLAUDE.local.md` | 본인만(현재 프로젝트) |

처음 시작한다면 **프로젝트 지침**(`./CLAUDE.md`)부터 만들어보세요. 프로젝트 루트에 파일 하나만 만들면 바로 사용할 수 있고, 팀원과도 공유할 수 있어 가장 효과적인 출발점입니다.

범위 개념이 낯설다면 [설정 범위(Scope)](/guides/environment-setup/scope)를 함께 참고하세요.

## CLAUDE.md 파일 생성

처음부터 직접 작성하지 않아도 됩니다. `/init` 명령어를 실행하면 Claude가 코드베이스를 분석해 CLAUDE.md를 자동으로 만들어줍니다.

<SlashBlock>
```
> /init
```
</SlashBlock>

자세한 사용법은 [/init 레퍼런스](/reference/commands/init)를 참고하세요.

Claude가 프로젝트를 살펴보고 빌드 명령, 테스트 지침, 프로젝트 규칙 등을 담은 파일을 생성합니다.

코드가 없는 빈 프로젝트에서 실행하면 분석할 내용이 없으므로, 기본 구조만 갖춘 최소한의 CLAUDE.md가 생성됩니다. 이 파일을 출발점 삼아 직접 내용을 채워나가면 됩니다.

CLAUDE.md가 이미 존재하면 `/init`은 파일을 덮어쓰지 않고 개선할 수 있는 사항을 제안합니다. Claude가 스스로 발견하지 못한 지침이 있다면 그 제안을 바탕으로 내용을 보완해나가세요.

> 실수를 발견할 때마다 CLAUDE.md에 한 줄 추가.<br/>처음부터 완벽하게 쓰려고 하지 마세요.
