# 스킬이란?

스킬은 Claude에게 **특정 작업이나 워크플로우를 처리하는 방법**을 가르치는 지침 모음이에요.\
마크다운 파일로 작성해 폴더에 넣어두면, Claude가 상황에 맞게 꺼내 써요.

매번 대화할 때마다 선호사항·프로세스·도메인 지식을 다시 설명하는 대신, 스킬로 한 번 가르쳐두면 이후로는 자동으로 적용돼요.

---

## 왜 스킬이 필요한가요?

Claude는 기본적으로 매 세션 비슷한 맥락에서 시작해요. 그런데 "코드 리뷰할 때는 이런 순서로 봐줘", "PR 만들 때는 이 양식 써줘" 같은 워크플로우를 매번 설명하는 건 번거로워요.

스킬을 정의해두면:

- 같은 작업을 매번 지시하지 않아도 돼요
- 팀원 모두가 동일한 방식으로 Claude를 활용할 수 있어요
- 복잡한 다단계 작업을 한 명령으로 실행할 수 있어요

---

## 스킬이 빛나는 순간

**반복 가능한 워크플로우**가 있을 때 스킬의 효과가 극대화돼요.

| 상황 | 예시 |
|------|------|
| UI 생성 | 사양서를 보고 팀 디자인 시스템에 맞는 컴포넌트 만들기 |
| 리서치 | 일관된 방법론으로 정보 수집·요약 |
| 문서 작성 | 팀 스타일 가이드를 따르는 기술 문서 작성 |
| 다단계 프로세스 | 기획 → 설계 → 구현 → 리뷰를 단계별로 조율 |

코드 실행·문서 작성 같은 Claude의 내장 기능과도 잘 맞아요.\
MCP를 함께 쓰면 원시 툴 접근을 신뢰할 수 있는 워크플로우로 전환하는 강력한 레이어가 추가돼요.

---

## 누구에게 필요한가요?

| 대상 | 이유 |
|------|------|
| 개발자 | Claude가 특정 워크플로우를 일관되게 따르게 하고 싶을 때 |
| 파워 유저 | 자신만의 작업 방식을 Claude에게 학습시키고 싶을 때 |
| 팀·조직 | 구성원 전체가 동일한 기준으로 Claude를 쓰게 표준화할 때 |

---

## Command vs Skill 핵심 차이

| | Command | Skill |
| :--- | :--- | :--- |
| **호출 방식** | 사용자가 `/명령어`로 직접 호출 | Claude가 상황 판단해서 자동 로드 |
| **용도** | 내가 실행하고 싶을 때 | Claude가 알아서 써야 할 때 |
| **인라인 bash** | ✅ 지원 (`!` 문법) | 지원하긴 하지만 주 목적 아님 |

---

#### 정리하면

- **Command** = 사람이 트리거 → 워크플로우 자동화
- **Skill** = Claude가 트리거 → 지식·능력 주입

---

## 스킬 호출 방법

스킬은 두 가지 방식으로 작동해요.

**방법 1 — 내가 직접 부르기**

슬래시 명령어로 호출하면 Claude가 해당 스킬의 지침에 따라 바로 작업을 시작해요.

<SlashBlock>
```
> /claude-md-improver
```
</SlashBlock>

**방법 2 — Claude가 알아서 꺼내 쓰기**

스킬은 Claude가 대화 흐름을 보고 <mark>필요하다고 판단하면 자동으로 로드</mark>하기도 해요. 내가 명령어를 외우지 않아도 돼요.

예를 들어 팀에 `code-review` 스킬이 설치되어 있다면, "리뷰해줘"라는 말 한 마디만으로 체계적인 리뷰가 시작돼요.

<VibeBlock>
```
PR 올렸는데 리뷰 한번 해줘
```
</VibeBlock>
