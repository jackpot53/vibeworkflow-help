# 세션 명령어 (Commands)

Claude Code는 자연어 대화 외에도 `/`로 시작하는 **슬래시 명령어**를 지원합니다.
명령어를 사용하면 설정 변경, 대화 관리, 파일 내보내기 등을 빠르게 실행할 수 있습니다.

---

## 명령어 사용법

터미널에서 `/`를 입력하면 사용 가능한 명령어 목록이 팝업으로 나타납니다.
`/` 뒤에 문자를 입력하면 해당 문자로 시작하는 명령어만 필터링됩니다.

예를 들어 `/c`를 입력하면 `/clear`, `/compact`, `/config`, `/context`, `/copy`, `/cost` 등이 표시됩니다.

<Callout type="info">
  **모든 명령어가 항상 표시되는 건 아닙니다.** 사용 중인 플랫폼, 요금제, 환경에 따라 표시되는 명령어가 다를 수 있습니다.

  - `/desktop`: macOS와 Windows에서만 표시됩니다
  - `/upgrade`, `/privacy-settings`: Pro 및 Max 요금제에서만 표시됩니다
  - `/terminal-setup`: 터미널이 키바인딩을 기본 지원하면 목록에서 숨겨집니다
</Callout>

기본 제공 명령어 외에도 `/simplify`, `/batch`, `/debug` 같은 [번들 스킬](/guides/skills)도 `/`를 입력할 때 함께 목록에 나타납니다.
나만의 명령어를 만들고 싶다면 [스킬](/guides/skills)을 참고하세요.

---

## 명령어 목록

아래 표에서 `<인수>`는 반드시 입력해야 하는 **필수 인수**, `[인수]`는 입력하지 않아도 되는 **선택 인수**를 나타냅니다.

| 명령어 | 설명 |
| :--- | :--- |
| `/add-dir <경로>` | 현재 세션에 작업 폴더 추가. Claude가 해당 폴더의 파일도 읽고 수정할 수 있게 됩니다 |
| `/agents` | [서브에이전트](/guides/subagents) 설정 관리 |
| `/btw <질문>` | 현재 대화 흐름에 영향 없이 빠른 부가 질문 하기 |
| `/chrome` | Chrome 브라우저와 Claude 연동 설정 |
| `/clear` | 대화 기록을 모두 지우고 컨텍스트 초기화. 별칭: `/reset`, `/new` |
| `/compact [지침]` | 긴 대화를 요약해 컨텍스트 용량 확보. 지침을 입력하면 특정 주제를 중심으로 요약 |
| `/config` | 테마, 모델, 출력 스타일 등 기본 설정 화면 열기. 별칭: `/settings` |
| `/context` | 현재 컨텍스트 사용량을 색상 그리드로 시각화해서 확인 |
| `/copy` | 마지막 Claude 응답을 클립보드에 복사. 코드 블록이 여러 개이면 어느 것을 복사할지 선택 가능 |
| `/cost` | 이번 세션에서 사용한 토큰 수와 비용 통계 표시 |
| `/desktop` | 현재 세션을 Claude Code 데스크톱 앱에서 이어서 진행. macOS·Windows 전용. 별칭: `/app` |
| `/diff` | 저장되지 않은 변경 사항과 턴별 diff를 보는 대화형 뷰어 열기. 방향키로 파일 탐색 및 전환 가능 |
| `/doctor` | Claude Code 설치 상태와 설정을 자동으로 진단·검사 |
| `/exit` | Claude Code 종료. 별칭: `/quit` |
| `/export [파일명]` | 현재 대화를 텍스트 파일로 내보내기. 파일명 없이 실행하면 클립보드 복사 또는 파일 저장 중 선택 |
| `/extra-usage` | 사용량 한도에 도달했을 때 추가로 사용할 수 있도록 설정 |
| `/fast [on\|off]` | 빠른 모드(Fast Mode) 켜기·끄기 |
| `/feedback [신고]` | Claude Code에 대한 피드백 제출 또는 버그 신고. 별칭: `/bug` |
| `/fork [이름]` | 이 시점에서 현재 대화의 복사본(포크)을 생성해 다른 방향으로 실험 가능 |
| `/help` | 도움말 및 사용 가능한 명령어 목록 표시 |
| `/hooks` | [훅(hook)](/guides/hooks) 설정 관리 |
| `/ide` | IDE 연동 관리 및 현재 연결 상태 표시 |
| `/init` | `CLAUDE.md` 파일을 생성해 프로젝트 초기화 |
| `/insights` | 세션 사용 패턴, 자주 작업한 영역, 불편했던 지점을 분석한 보고서 생성 |
| `/install-github-app` | 리포지토리에 Claude GitHub Actions 앱 설치 및 설정. 리포지토리 선택과 연동 과정을 단계별로 안내 |
| `/install-slack-app` | Claude Slack 앱 설치. 브라우저에서 OAuth 인증 완료 |
| `/keybindings` | 단축키 설정 파일 열기 또는 새로 생성 |
| `/login` | Anthropic 계정 로그인 |
| `/logout` | Anthropic 계정 로그아웃 |
| `/mcp` | MCP 서버 연결 및 OAuth 인증 관리 |
| `/memory` | `CLAUDE.md` 메모리 파일 편집, 자동 메모리 켜기·끄기, 저장된 메모리 항목 확인 |
| `/mobile` | Claude 모바일 앱 다운로드 QR 코드 표시. 별칭: `/ios`, `/android` |
| `/model [모델명]` | 사용할 AI 모델 선택 또는 변경. 방향키로 노력 수준도 조절 가능. 변경 사항은 즉시 적용됩니다 |
| `/passes` | 친구에게 Claude Code 1주일 무료 이용권 공유. 계정이 해당 조건을 충족할 때만 표시 |
| `/permissions` | 도구 실행 권한 보기 또는 업데이트. 별칭: `/allowed-tools` |
| `/plan` | 프롬프트에서 바로 Plan Mode(계획 모드)로 진입 |
| `/plugin` | Claude Code [플러그인](/guides/plugins) 관리 |
| `/pr-comments [PR]` | GitHub 풀 리퀘스트의 댓글 가져오기. 현재 브랜치 PR 자동 감지 또는 PR URL·번호 직접 입력 가능. `gh` CLI가 필요합니다 |
| `/privacy-settings` | 개인정보 보호 설정 보기 및 업데이트. Pro·Max 요금제 전용 |
| `/release-notes` | 전체 변경 내역(changelog) 보기. 최신 버전이 맨 위에 표시됩니다 |
| `/reload-plugins` | 활성화된 모든 [플러그인](/guides/plugins)을 다시 불러와 재시작 없이 변경 사항 적용. 재시작이 필요한 변경 사항은 따로 안내됩니다 |
| `/remote-control` | 현재 세션을 claude.ai에서 원격으로 제어할 수 있도록 설정. 별칭: `/rc` |
| `/remote-env` | 웹 텔레포트 세션의 기본 원격 환경 설정 |
| `/rename [이름]` | 현재 세션 이름 바꾸기. 이름 없이 실행하면 대화 내용을 바탕으로 자동 생성 |
| `/resume [세션]` | ID 또는 이름으로 이전 대화 재개하거나 세션 선택기 열기. 별칭: `/continue` |
| `/review` | ⚠️ **더 이상 지원되지 않습니다.** 대신 `code-review` 플러그인을 설치하세요: `claude plugin install code-review@claude-code-marketplace` |
| `/rewind` | 대화와 코드를 이전 시점으로 되돌리거나, 선택한 메시지 이후를 요약. 별칭: `/checkpoint` |
| `/sandbox` | 샌드박스 모드 켜기·끄기. 지원되는 플랫폼에서만 사용 가능 |
| `/security-review` | 현재 브랜치의 변경 사항에서 보안 취약점(주입 공격, 인증 문제, 데이터 노출 등)을 자동 분석 |
| `/skills` | 사용 가능한 [스킬](/guides/skills) 목록 보기 |
| `/stats` | 일일 사용량, 세션 기록, 연속 사용 기록, 모델 사용 현황 시각화 |
| `/status` | 버전, 모델, 계정, 연결 상태를 보여주는 설정 화면(상태 탭) 열기 |
| `/statusline` | 셸 프롬프트에 표시할 Claude Code 상태 줄 설정. 원하는 내용을 설명하거나 인수 없이 실행하면 자동 구성 |
| `/stickers` | Claude Code 스티커 주문 |
| `/tasks` | 백그라운드 작업 목록 보기 및 관리 |
| `/terminal-setup` | Shift+Enter 등 단축키를 위한 터미널 키바인딩 설정. VS Code, Alacritty, Warp 등 설정이 필요한 터미널에서만 표시 |
| `/theme` | 색상 테마 변경. 밝은·어두운 테마, 색맹 접근성(daltonized) 테마, 터미널 색상 팔레트를 활용하는 ANSI 테마 포함 |
| `/upgrade` | 더 높은 요금제로 업그레이드하는 페이지 열기 |
| `/usage` | 현재 요금제의 사용 한도와 속도 제한 상태 확인 |
| `/vim` | Vim 편집 모드와 일반 편집 모드 간 전환 |
