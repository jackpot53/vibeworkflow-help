# 내장 도구 (Built-in Tools)

Claude Code는 터미널에서 동작하는 AI 어시스턴트로, **도구(Tool)**를 통해 파일을 읽고, 코드를 편집하고, 명령어를 실행하는 등 실질적인 작업을 수행합니다.

사용자가 직접 도구를 호출할 필요는 없습니다. "인증 관련 버그 고쳐줘"처럼 자연어로 요청하면 Claude가 알아서 적절한 도구를 선택해 사용합니다.

---

## Claude가 사용할 수 있는 도구

| 도구 | 설명 | 권한 필요 |
| :--- | :--- | :---: |
| **Agent** | 복잡한 작업을 병렬로 처리할 때 독립된 AI 하위 작업자(서브에이전트)를 생성합니다 | — |
| **AskUserQuestion** | 작업 요청이 모호할 때 객관식으로 추가 질문을 합니다 | — |
| **Bash** | 터미널 명령어를 실행합니다 (`npm test`, `git commit` 등) | ✓ |
| **CronCreate** | 현재 세션 안에서 반복 또는 예약 작업을 등록합니다 (세션 종료 시 사라짐) | — |
| **CronDelete** | 등록된 예약 작업을 취소합니다 | — |
| **CronList** | 현재 세션의 예약 작업 목록을 조회합니다 | — |
| **Edit** | 파일의 일부를 수정합니다 (파일 전체 재작성 없이 특정 부분만 변경) | ✓ |
| **EnterPlanMode** | 코드 작성 전에 접근 방식을 설계하는 계획 모드로 전환합니다 | — |
| **EnterWorktree** | 기존 브랜치에 영향 없이 작업할 수 있는 독립된 git 작업 공간을 만듭니다 | — |
| **ExitPlanMode** | 작성한 계획을 사용자에게 제시하고 계획 모드를 종료합니다 | ✓ |
| **ExitWorktree** | 독립 작업 공간을 닫고 원래 디렉토리로 돌아갑니다 | — |
| **Glob** | 패턴으로 파일을 검색합니다 (`*.ts`, `src/**/*.css` 등) | — |
| **Grep** | 파일 내용에서 특정 텍스트나 패턴을 검색합니다 | — |
| **ListMcpResourcesTool** | 연결된 MCP 서버에서 사용 가능한 리소스 목록을 가져옵니다 | — |
| **LSP** | 언어 서버를 통해 타입 오류·경고 확인, 정의 이동, 참조 찾기 등 코드 인텔리전스 기능을 제공합니다. 별도 코드 인텔리전스 플러그인이 필요합니다 | — |
| **NotebookEdit** | Jupyter 노트북(.ipynb) 파일의 셀을 수정합니다 | ✓ |
| **Read** | 파일 내용을 읽습니다 | — |
| **ReadMcpResourceTool** | MCP 서버의 특정 리소스를 주소(URI)로 읽어옵니다 | — |
| **Skill** | 대화 안에서 스킬(사전 정의된 작업 묶음)을 실행합니다 | ✓ |
| **TaskCreate** | 작업 목록에 새 항목을 추가합니다 | — |
| **TaskGet** | 특정 작업의 상세 내용을 조회합니다 | — |
| **TaskList** | 전체 작업 목록과 현재 상태를 조회합니다 | — |
| **TaskOutput** | 백그라운드에서 실행 중인 작업의 출력 결과를 가져옵니다 | — |
| **TaskStop** | 실행 중인 백그라운드 작업을 중단합니다 | — |
| **TaskUpdate** | 작업 상태·내용·종속성을 수정하거나 작업을 삭제합니다 | — |
| **TodoWrite** | 세션 중 처리할 항목의 체크리스트를 관리합니다. 비대화형 모드(헤드리스)와 Agent SDK에서 사용되며, 일반 대화 세션에서는 Task 계열 도구를 씁니다 | — |
| **ToolSearch** | 도구 검색이 활성화된 환경에서 지연 로드된 도구를 검색·불러옵니다 | — |
| **WebFetch** | 지정한 URL의 웹 페이지 내용을 가져옵니다 | ✓ |
| **WebSearch** | 웹 검색을 수행합니다 | ✓ |
| **Write** | 새 파일을 만들거나 기존 파일 전체를 덮어씁니다 | ✓ |

<Callout type="info">
  **권한 필요(✓)** 표시가 있는 도구는 Claude가 실행하기 전에 사용자에게 확인을 요청합니다. 자주 쓰는 도구는 [권한 설정](/guides/environment-setup/settings#권한-설정)에서 자동 허용으로 등록해 두면 매번 승인하지 않아도 됩니다.
</Callout>

## 도구는 어떻게 쓰이나요?

내장 도구는 **사용자가 직접 호출하는 게 아니라, 자연어로 작업을 요청하면 Claude가 자동으로 선택해서 사용**합니다.

예를 들어 이렇게 요청하면:

<VibeBlock>
```
이 프로젝트에서 인증 관련 코드 찾아줘
```
</VibeBlock>

Claude가 내부적으로 Grep, Glob, Read 등을 알아서 조합해 실행합니다. "Grep 도구를 써서 검색해줘"라고 할 필요가 없습니다.

새 기능을 추가할 때 Claude는 보통 이런 흐름으로 작업합니다:

1. **Glob** — 관련 파일 탐색
2. **Read** — 기존 코드 파악
3. **Edit / Write** — 코드 작성·수정
4. **Bash** — 테스트 실행 및 확인

### 요청 방식에 따른 도구 선택

| 이렇게 말하면 | Claude가 사용하는 도구 |
| :--- | :--- |
| "src 폴더에서 .ts 파일 목록 보여줘" | **Glob** |
| "handleAuth 함수 어디 있어?" | **Grep** |
| "config.json 내용 보여줘" | **Read** |
| "이 함수 이름 바꿔줘" | **Read → Edit** |
| "새 컴포넌트 파일 만들어줘" | **Write** |
| "테스트 돌려봐" | **Bash** (`npm test` 등) |
| "최신 React 문서 확인해줘" | **WebFetch** |
| "이 작업 단계별로 계획 세워줘" | **TodoWrite** |
| "코드베이스 전체 분석해줘" | **Agent** (서브에이전트) |

<Callout type="tip">
  사용자의 역할은 도구를 직접 호출하는 것이 아니라, **어떤 도구를 허용할지 권한을 설정하는 것**입니다. 나머지는 Claude가 알아서 처리합니다.
</Callout>

## 도구, 앞으로 이런 곳에서 다시 만납니다

지금까지 살펴본 도구들은 이 가이드의 다음 주제들과 직접 연결됩니다. 각 단계에서 도구를 어떻게 활용하는지 하나씩 살펴보겠습니다.

| 주제 | 도구와의 연결 | 가이드 |
| :--- | :--- | :--- |
| **스킬** | 스킬을 만들 때 `allowed-tools`로 해당 스킬이 쓸 수 있는 도구를 직접 지정합니다 | [스킬(SKILL)](/guides/skills) |
| **서브에이전트** | 서브에이전트를 정의할 때 각 에이전트가 접근할 수 있는 도구를 제한할 수 있습니다 | [서브에이전트](/guides/subagents) |
| **훅** | 도구 실행 전후에 자동으로 실행할 명령을 등록해 도구의 동작을 확장할 수 있습니다 | [훅(HOOK)](/guides/hooks) |
| **권한 설정** | 어떤 도구를 자동 허용·확인·차단할지 `settings.json`에서 세밀하게 제어합니다 | [설정](/guides/environment-setup/settings#권한-설정) |

<Callout type="info">
  일반적인 사용에서는 "이 버그 고쳐줘"처럼 자연어로 요청하고 Claude가 도구를 알아서 선택하게 하는 것이 기본 패턴입니다. 도구를 직접 지정하거나 제한하는 것은 스킬·서브에이전트 설계, 보안 설정, 특수한 워크플로우를 구성할 때 주로 사용됩니다.
</Callout>
