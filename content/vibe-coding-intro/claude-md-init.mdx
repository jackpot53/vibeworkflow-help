# CLAUDE.md 초기화하기

![CLAUDE.md 초기화하기](/images/medium-git-diagram.jpeg)

## CLAUDE.md가 왜 필요한가요?

신입사원이 첫 출근을 하면 회사에서 온보딩 자료를 건네주죠.\
"우리 팀은 이 툴 써요, 보고는 이렇게 해요, 이 파일 건드리면 안 돼요, 점심은 12시 반이에요, 배포는 금요일엔 하지 마요, 고객 데이터는 함부로 수정하지 마요" 같은 것들이요.

만약 온보딩 자료가 없다면 어떻게 될까요?\
신입사원이 출근할 때마다 사수에게 같은 걸 물어봐야 하겠죠.\
"이거 어떻게 하는 거예요?", "이 파일 제가 수정해도 돼요?", "보고는 어디에 올려요?"...\
아마 사수 짜증이 폭발할 거예요.

클로드코드도 마찬가지예요.\
새 대화를 열 때마다 아무것도 모르는 상태로 시작하거든요.\
즉, 이전에 "이렇게 해줘", "저건 하지 마", "이 스타일로 작성해줘" 라고 했던 반복적인 규칙이나 지침들을 다 잊어버려요.

그래서 신입사원의 온보딩 자료가 필요한 거죠.\
그 역할을 하는 게 바로 **`CLAUDE.md`** 파일이에요.\
클로드코드가 프로젝트 폴더에서 실행될 때마다 이 파일을 자동으로 읽기 때문에,\
**한 번 적어두면 매번 설명하지 않아도** Claude가 맥락을 이해하고 바로 작업을 시작해요.

---

## 어디에 만드나요?

프로젝트 **루트 폴더** (가장 바깥 폴더)에 `CLAUDE.md`라는 이름으로 저장해요.\
예를 들어 `portfolio` 폴더에서 프로젝트를 만들고 있다면, 그 폴더 안에 만들면 돼요.

```
portfolio/          ← 여기가 루트 폴더 (프로젝트 폴더)
├── CLAUDE.md       ← 여기에 만들어요
```

<Callout type="info">
  파일 이름은 반드시 **대문자**로 `CLAUDE.md` 이에요. 소문자로 쓰면 클로드코드가 인식하지 못해요.
</Callout>

---

## 어떤 내용을 담나요?

아래 정보를 담아두면 Claude가 처음 프로젝트를 봐도 바로 맥락을 이해하고 작업할 수 있어요.

| 항목 | 왜 필요한가요? |
|---|---|
| **실행 명령어** | Claude가 직접 서버를 켜거나 빌드할 때 필요해요. `npm run dev`, 빌드, 테스트 명령어를 적어두세요. |
| **기술 스택과 버전** | "Next.js 15 App Router" 처럼 구체적으로 적어야 Claude가 맞는 문법과 패턴을 써요. |
| **폴더 구조** | 컴포넌트가 어디 있는지, 어떤 폴더가 어떤 역할인지 알아야 파일을 정확히 찾아요. |
| **건드리면 안 되는 것** | 삭제 금지 파일, 수정하면 안 되는 설정, 특정 패턴 금지 사항을 명시해요. |
| **배포 환경** | Vercel인지 Railway인지, 어느 브랜치에서 배포하는지 적어두면 Claude가 실수를 피해요. |
| **코딩 스타일** | 주석 언어, 네이밍 규칙, 컴포넌트 작성 방식 등 일관성을 유지하는 데 필요해요. |

낯선 키워드가 많을 수 있어요. 지금 다 이해하지 않아도 괜찮아요. 바이브코딩을 하다 보면 자연스럽게 알게 돼요.

그래서 `CLAUDE.md` 파일은 대충 이렇게 생겼어요.

<FileBlock filename="CLAUDE.md">
```markdown
# 프로젝트 개요
나만의 포트폴리오 사이트. Next.js로 만들었고 Vercel에 배포되어 있어요.

## 기술 스택
- Next.js 15 (App Router)
- Tailwind CSS
- Vercel 배포

## 자주 쓰는 명령어
- `npm run dev` — 개발 서버 시작 (http://localhost:3000)
- `npm run build` — 배포용 빌드

## 주의사항
- `public/` 폴더의 이미지는 직접 올린 것이라 함부로 삭제하지 마세요
- 배포는 항상 main 브랜치에서 해요

## 코딩 스타일
- 컴포넌트 이름은 한국어 주석으로 설명해줘요
- 코드 설명은 항상 한국어로 해줘요
```
</FileBlock>

이 파일을 읽은 Claude에게 이렇게 말해봐요.

<VibeBlock>
```
메인 페이지에 "지금 시작하기" 버튼 추가해줘
```
</VibeBlock>

그러면 Claude는 CLAUDE.md를 읽었기 때문에 따로 설명 안 해도 이미 알고 있어요.\
이게 포트폴리오 사이트라는 것, Tailwind CSS로 스타일을 잡는다는 것, 설명은 한국어로 해야 한다는 것, `public/` 폴더 이미지는 건드리면 안 된다는 것까지요.

<ClaudeBlock>
```
app/page.tsx 파일에 버튼을 추가할게요.

Tailwind CSS를 사용 중이니 클래스로 스타일링할게요.
코드 설명은 한국어로 달아드릴게요.

// 메인 페이지 시작 버튼
<button className="bg-blue-500 hover:bg-blue-600 text-white px-6 py-3 rounded-lg">
  지금 시작하기
</button>
```
</ClaudeBlock>

"Tailwind CSS 써요"라고 말 안 해도 알고, "한국어로 설명해줘"라고 말 안 해도 알고, 어느 파일을 수정해야 하는지도 알아요.\
`CLAUDE.md`에 적어뒀으니까요.

<Callout type="tip">
  <span className="inline-block text-xs font-semibold tracking-widest uppercase px-2 py-0.5 mb-2 bg-emerald-500/15 text-emerald-600 dark:text-emerald-400 border border-emerald-500/30">Best Practice</span>
  처음부터 완벽하게 적을 필요는 없어요.
  쓰다 보면 "이건 계속 설명해야 하네" 싶은 게 생기면 그때 추가하면 돼요.
</Callout>

## `/init` 명령어로 자동 생성하기

직접 작성하기 어렵다면 클로드코드의 `/init` 명령어를 사용하면 돼요.\
Claude가 프로젝트 폴더를 스스로 분석해서 초안을 자동으로 만들어줘요. 그 후 본인이 내용을 다듬으면 끝이에요.

<Tabs items={['1. claude 실행', '2. /init 입력', '3. 파일 확인 & 수정']}>
  <Tabs.Tab>

터미널을 열고 프로젝트 폴더로 이동한 뒤 `claude`를 실행해요.

<ShellBlock>
```bash
cd dev/portfolio
claude
```
</ShellBlock>

<Callout type="info">
  반드시 **프로젝트 폴더 안에서** `claude`를 실행해야 해요.\
  그래야 Claude가 이 폴더의 코드를 분석할 수 있어요.
</Callout>

  </Tabs.Tab>
  <Tabs.Tab>

클로드코드가 열리면 `/init`을 입력하고 Enter를 눌러요.

<SlashBlock>
```
/init
```
</SlashBlock>

Claude가 폴더 안의 파일들을 살펴보고 `CLAUDE.md` 초안을 작성해줘요.\
생성이 완료되면 "Created CLAUDE.md" 같은 메시지가 나타나요.

  </Tabs.Tab>
  <Tabs.Tab>

생성된 `CLAUDE.md` 파일을 열어서 내용을 확인해요.

자동 생성된 내용에 이런 것들을 **추가**해두면 더 유용해요:

<ul>
  <li>나만의 코딩 스타일 (예: "설명은 항상 한국어로")</li>
  <li>배포 관련 정보 (예: "Vercel로 배포, main 브랜치 기준")</li>
  <li>건드리지 말아야 할 파일이나 폴더</li>
  <li>자주 까먹는 명령어나 주의사항</li>
</ul>

  </Tabs.Tab>
</Tabs>

<Callout type="info">
  `/init` 명령어의 실행 흐름, 생성 파일 예시, 주의사항 등 자세한 내용은 [/init 명령어 레퍼런스](/reference/commands/init)를 참고하세요.
</Callout>

## CLAUDE.md 관리 팁

**자주 업데이트하세요**\
프로젝트가 바뀌면 `CLAUDE.md`도 함께 업데이트해요. 오래된 정보는 Claude를 혼란스럽게 할 수 있어요.

**Claude에게 직접 부탁해도 돼요**\
새로운 정보가 생기면 클로드코드에서 이렇게 말해보세요:

<VibeBlock>
```
CLAUDE.md에 "버튼 색상은 항상 브랜드 컬러인 파란색으로 통일해줘"라는 내용 추가해줘
```
</VibeBlock>

**팀 작업이라면 GitHub에 올려요**\
`CLAUDE.md`는 일반 파일이라 GitHub에 올려두면 팀원 모두가 같은 설정으로 Claude를 쓸 수 있어요.

  [클로드코드 가이드 → CLAUDE.md](/guides/claude-md)에서 확인할 수 있어요.
