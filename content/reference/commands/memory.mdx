## 언제 쓰나요?

Claude가 현재 어떤 정보를 기억하고 있는지 확인하거나, 자동 기억 기능을 켜고 끄고 싶을 때 사용해요.

**이런 상황에서 쓰세요:**

- Claude가 이 세션에서 어떤 CLAUDE.md 파일을 읽고 있는지 확인하고 싶을 때
- 자동 기억(auto memory) 기능을 켜거나 끄고 싶을 때
- 자동 기억이 저장된 폴더를 열어보고 싶을 때

---

## 사용법

```
> /memory
```

```
Auto-memory: on

    1. Project memory             Checked in at ./CLAUDE.md
    2. User memory                Saved in ~/.claude/CLAUDE.md
  ❯ 3. Open auto-memory folder ✔
```

실행하면 다음을 확인하고 관리할 수 있어요.

- 현재 세션에 로드된 **CLAUDE.md 및 rules 파일 목록**
- **자동 기억(auto memory) 켜기·끄기** 토글
- 자동 기억이 저장된 폴더로 이동하는 링크

## Claude Code의 두 가지 기억 시스템

Claude Code는 두 가지 기억 시스템을 함께 사용해요. 둘 다 매 세션 시작 시 로드됩니다.

### 1. CLAUDE.md — 내가 직접 작성하는 기억

Claude의 행동을 지속적으로 안내하는 마크다운 파일이에요. 여러 위치에 계층적으로 관리할 수 있고, 모두 누적 적용됩니다.

| 위치 | 파일 경로 | 적용 범위 |
|------|-----------|-----------|
| 글로벌 | `~/.claude/CLAUDE.md` | 모든 프로젝트에 적용 |
| 프로젝트 | 프로젝트 루트의 `CLAUDE.md` | 해당 프로젝트에만 적용 |
| 하위 디렉토리 | 중첩 폴더의 `CLAUDE.md` | 해당 폴더 작업 시 추가 적용 |

CLAUDE.md에 넣으면 좋은 내용:
- 빌드·테스트 명령어
- 코딩 컨벤션 및 스타일 규칙
- 아키텍처 결정 사항
- 작업 워크플로우 규칙

### 2. Auto Memory — Claude가 스스로 쌓는 기억

Claude가 작업하면서 유용한 정보를 자동으로 저장해요. 빌드 명령어, 디버깅 패턴, 아키텍처 결정, 코드 스타일 등을 스스로 판단해서 기록합니다.

저장 위치는 `~/.claude/projects/<project>/memory/`이며, 구조는 다음과 같아요.

```
~/.claude/projects/<project>/memory/
├── MEMORY.md          # 간결한 인덱스 (최대 200줄, 매 세션 로드)
├── debugging.md       # 디버깅 패턴 상세 노트
├── api-conventions.md # API 설계 결정
└── ...
```

`MEMORY.md`는 인덱스 역할로 간결하게 유지되고, 상세한 내용은 별도 토픽 파일로 분리돼요. 토픽 파일은 세션 시작 시 자동으로 로드되지 않고, Claude가 필요하다고 판단할 때 직접 읽습니다.

<Callout type="info">
  Auto Memory가 꺼져 있으면 Claude는 세션 간에 아무것도 자동으로 저장하지 않아요.\
  수동으로 CLAUDE.md를 관리하고 싶다면 꺼두는 것도 좋은 선택이에요.
</Callout>
