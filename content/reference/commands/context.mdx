## 언제 쓰나요?

Claude가 느려지거나, "컨텍스트가 거의 꽉 찼다"는 경고를 받기 전에 현재 상태를 미리 확인하고 싶을 때 사용해요.

---

## 컨텍스트 창이란?

Claude가 한 번의 대화에서 "기억"할 수 있는 텍스트의 최대 양이에요.\
대화 내용, 읽어들인 파일, 도구 실행 결과 등이 모두 여기에 쌓여요.

비유하자면 Claude의 **단기 기억 용량**이에요. 가득 차면 오래된 내용부터 잊어버리거나, 자동으로 요약이 일어나요.

<Callout type="info">
  컨텍스트 창 크기는 모델마다 달라요.\
  Claude Sonnet 4.6은 20만 토큰, Claude Opus 4.7은 100만 토큰이에요.
</Callout>

---

## 사용법

```
> /context
```

실행하면 색상으로 구분된 블록 형태로 사용량이 시각적으로 표시돼요:

```
● Context Window
  ████████████░░░░░░░░  58% used
  116,000 / 200,000 tokens
```

---

## 사용률에 따라 어떻게 행동하면 될까요?

| 사용률 | 상태 | 권장 행동 |
|--------|------|-----------|
| ~60% | 정상 | 계속 작업 |
| 70~80% | 주의 | `/compact`로 요약 권장 |
| 90% 이상 | 위험 | 즉시 `/compact` 또는 `/clear` 실행 |
| 100% | 한도 초과 | Claude Code가 자동으로 요약 실행 |

<Callout type="tip">
  자동 요약이 일어나기 전에 직접 `/compact`를 실행하면\
  중요한 내용이 더 잘 보존돼요.
</Callout>

---

## 관련 명령어

| 명령어 | 설명 |
|--------|------|
| `/usage` | 토큰 수·비용 등 수치로 상세 확인 |
| `/compact` | 대화 요약해서 공간 확보 |
| `/clear` | 대화 전체 삭제 |
