## 언제 쓰나요?

테마, 출력 스타일, 자동 compact 같은 설정을 세션 중에 바로 바꾸고 싶을 때 사용해요.

**이런 상황에서 쓰세요:**

- Claude 응답 포맷이 마음에 안 들어서 출력 스타일을 바꾸고 싶을 때
- 자동 compact가 너무 자주 또는 늦게 일어나서 임계값을 조정하고 싶을 때
- 현재 어떤 설정이 적용되어 있는지 확인하고 싶을 때

---

## 사용법

```
> /config
```

실행하면 설정 항목이 메뉴 형태로 나타나고, 방향키로 선택해서 변경할 수 있어요.

## 설정 파일 직접 편집

`/config` 메뉴에서 제공하지 않는 세부 설정은 `~/.claude/settings.json`을 직접 편집해서 관리할 수 있어요.

```json
{
  "model": "claude-sonnet-4-6",
  "permissions": {
    "allow": ["Bash(npm run lint)", "Bash(npm run test *)"],
    "deny": ["Read(./.env)", "Read(./.env.*)"]
  },
  "env": {
    "CLAUDE_AUTOCOMPACT_PCT_OVERRIDE": "75"
  }
}
```

| 항목 | 설명 |
|------|------|
| `model` | 기본으로 사용할 모델 지정 |
| `permissions` | 허용/차단할 도구 및 명령어 |
| `hooks` | 특정 시점에 자동 실행할 셸 명령어 |
| `env` | 세션 간 유지할 환경 변수 |

<Callout type="info">
  **설정 파일 위치**

  - `~/.claude/settings.json` — 내 컴퓨터 전체에 적용되는 개인 설정
  - `.claude/settings.json` — 현재 프로젝트에만 적용되는 설정
</Callout>

## 설정 우선순위

여러 곳에 설정이 있을 때는 더 좁은 범위의 설정이 우선 적용돼요.

| 우선순위 | 설정 위치 |
|----------|-----------|
| 1 (최우선) | 조직 정책 (엔터프라이즈 관리자 설정) |
| 2 | 프로젝트 설정 (`.claude/settings.json`) |
| 3 | 개인 설정 (`~/.claude/settings.json`) |

<Callout type="warning">
  상위 수준 설정이 하위를 덮어씁니다.\
  조직 정책으로 막힌 설정은 개인 설정에서 바꿔도 적용되지 않아요.
</Callout>

## 관련 명령어

| 명령어 | 설명 |
|--------|------|
| `/model` | 사용할 AI 모델 변경 |
| `/compact` | 대화를 수동으로 요약·압축 |
| `/permissions` | 도구 실행 권한 확인 및 수정 |
| `/status` | 현재 세션 상태 및 설정 확인 |
