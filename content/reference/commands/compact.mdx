## 언제 쓰나요?

작업이 길어지면서 대화가 많이 쌓였는데, 지금까지의 흐름을 유지한 채로 계속 이어가고 싶을 때 사용해요.

**이런 상황에서 쓰세요:**

- 한 작업을 오래 하다 보니 대화가 수십 개 이상 쌓였을 때
- Claude가 "컨텍스트가 거의 꽉 찼다"고 경고할 때
- 속도가 느려지거나 이전 내용을 자꾸 까먹는 것 같을 때

---

## 사용법

인자 없이 실행하면 Claude가 알아서 중요한 내용 위주로 요약해요:

```
> /compact
```

요약할 때 무엇을 중점으로 남길지 직접 알려줄 수도 있어요:

```
> /compact 인증 관련 변경 사항 위주로 요약해줘
> /compact 지금 작업 중인 버그 수정 내용만 남겨줘
```

<Callout type="tip">
  `/compact` 직후에 "지금까지 뭘 했는지 요약해줘"라고 물어보면\
  Claude가 어떤 내용을 기억하고 있는지 확인할 수 있어요.
</Callout>

---

## 자동 컴팩션

컨텍스트 한도에 가까워지면 Claude Code가 자동으로 대화를 압축해요.\
이때 Claude는 작업을 잠깐 멈추고 이런 메시지를 보여줘요:

```
⚠ Context window is nearly full. Auto-compacting conversation...
```

자동으로 압축되기 전에 `/compact`를 직접 실행하면 중요한 내용이 더 잘 보존돼요.

---

## `/clear`와 뭐가 달라요?

| | `/compact` | `/clear` |
|--|------------|----------|
| 대화 기록 | 요약해서 보존 | 완전히 삭제 |
| 지금까지의 맥락 | 요약본으로 유지됨 | 사라짐 |
| 언제 쓰나 | 작업을 이어갈 때 | 새 작업 시작할 때 |

---

## 관련 명령어

| 명령어 | 설명 |
|--------|------|
| `/clear` | 대화 내용 완전 삭제 |
| `/context` | 현재 컨텍스트 사용량 확인 |
| `/usage` | 토큰 사용량 및 비용 확인 |
