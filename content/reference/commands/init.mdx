## 언제 쓰나요?

프로젝트를 처음 시작할 때, 또는 기존 프로젝트에 `CLAUDE.md`가 없을 때 사용해요.\
Claude가 코드 파일들을 스스로 읽어보고 프로젝트 구조·기술 스택·명령어 등을 정리한 초안을 만들어줘요.

<Callout type="info">
  `CLAUDE.md`는 클로드코드가 세션을 시작할 때 자동으로 읽는 파일이에요.\
  프로젝트 정보를 적어두면 매번 설명하지 않아도 Claude가 맥락을 이해한 채로 작업해요.\
  자세한 내용은 [CLAUDE.md 초기화하기](/getting-started/vibe-coding-intro/claude-md-init)를 참고하세요.
</Callout>

---

## 사용법

프로젝트 폴더에서 `claude`를 실행한 뒤 입력해요.

<SlashBlock>
```
> /init
```
</SlashBlock>

<Callout type="warning">
  반드시 **프로젝트 폴더 안에서** `claude`를 실행해야 해요.\
  다른 폴더에서 실행하면 엉뚱한 내용이 생성될 수 있어요.
</Callout>

---

## 실행 흐름

**1. Claude가 폴더를 분석해요**

`package.json`, 소스 파일, 설정 파일 등을 읽고 프로젝트의 구조와 기술 스택을 파악해요.

**2. CLAUDE.md 초안이 생성돼요**

프로젝트 루트(가장 바깥 폴더)에 `CLAUDE.md` 파일이 만들어지고, 터미널에 이런 메시지가 나타나요:

```
✓ Created CLAUDE.md
```

**3. 내용을 직접 다듬어요**

자동 생성된 내용을 열어보고, 빠진 내용을 추가하거나 틀린 내용을 고쳐요.

---

## 생성된 파일 예시

`/init`을 실행하면 이런 형태의 파일이 만들어져요:

<FileBlock filename="CLAUDE.md">
```markdown
# Project Overview
포트폴리오 웹사이트. Next.js로 만들었으며 Vercel에 배포됩니다.

## Tech Stack
- Next.js 15 (App Router)
- Tailwind CSS
- TypeScript

## Common Commands
- `npm run dev` — 개발 서버 시작
- `npm run build` — 프로덕션 빌드
- `npm run lint` — 코드 검사
```
</FileBlock>

<Callout type="tip">
  자동 생성된 내용에 이런 것들을 **추가**해두면 Claude가 더 정확하게 도와줘요:

  - 나만의 코딩 스타일 (예: "설명은 항상 한국어로 해줘")
  - 배포 환경 (예: "배포는 Vercel, main 브랜치 기준")
  - 건드리지 말아야 할 파일이나 폴더
  - 커밋 메시지 형식이나 브랜치 규칙
</Callout>

## 빈 프로젝트에서 실행할 때

분석할 코드가 없는 빈 폴더라면 Claude가 먼저 확인을 요청해요.

<SlashBlock>
```
> /init
```
</SlashBlock>

<ClaudeBlock>
```
⏺ The directory appears to be empty. There's no code to analyze, so there's
  nothing to document in a CLAUDE.md.

  Would you like me to initialize a minimal CLAUDE.md as a placeholder for when
  you add code to this project?
```
</ClaudeBlock>

`yes`를 입력하면 최소한의 CLAUDE.md가 생성돼요.

<VibeBlock>
```
> yes
```
</VibeBlock>

생성된 파일은 기본 안내 문구만 담은 최소한의 형태예요:

<FileBlock filename="CLAUDE.md">
```markdown
# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.
```
</FileBlock>

이 파일을 출발점 삼아 프로젝트에 맞게 내용을 직접 채워나가면 돼요.

---

## 이미 CLAUDE.md가 있을 때

이미 파일이 있는 경우 `/init`을 실행하면 덮어쓸지 물어봐요.\
기존 내용이 사라질 수 있으니, 중요한 내용은 미리 백업해두세요.
