## 언제 쓰나요?

Claude Code는 파일 수정, 명령어 실행 같은 작업을 할 때마다 기본적으로 확인을 요청합니다.
자주 쓰는 작업을 매번 승인하는 게 번거롭거나, 반대로 특정 파일이나 명령어는 절대 건드리지 못하게 막고 싶을 때 사용해요.

**이런 상황에서 쓰세요:**

- `git` 명령어나 `npm run lint` 같이 자주 쓰는 명령어를 매번 승인하는 게 귀찮을 때
- `.env` 같은 민감한 파일을 Claude가 절대 읽거나 수정하지 못하게 막고 싶을 때
- 현재 어떤 권한이 허용·차단되어 있는지 확인하고 싶을 때

---

## 사용법

```
> /permissions
```

실행하면 현재 권한 설정을 확인하고 수정할 수 있는 대화형 메뉴가 열립니다.

---

## 권한의 3단계

Claude Code의 모든 도구 실행은 다음 세 가지 중 하나로 처리됩니다.

| 단계 | 설명 |
|------|------|
| **allow** (허용) | Claude가 확인 없이 자동으로 실행 |
| **ask** (확인) | 실행 전에 사용자에게 허가 요청 (기본값) |
| **deny** (차단) | 항상 거부, 절대 실행 불가 |

규칙은 **deny → allow → ask** 순서로 평가됩니다. deny에 해당하면 즉시 차단되고, allow에 해당하면 자동 실행되며, 둘 다 해당하지 않으면 ask(확인 요청)가 기본 동작입니다.

---

## 설정 방법

권한 규칙은 `settings.json`의 `permissions` 항목에 직접 작성하거나, 세션 중 Claude가 권한을 요청할 때 **"항상 허용(Always allow)"** 을 선택하면 자동으로 추가됩니다.

<FileBlock filename="settings.json">
```json
{
  "permissions": {
    "allow": [
      "Read",
      "Bash(git *)",
      "Bash(npm run lint)",
      "Bash(npm run test *)"
    ],
    "deny": [
      "Read(./.env)",
      "Read(./.env.*)",
      "Write(./production.config.*)"
    ]
  }
}
```
</FileBlock>

<Callout type="info">
  `settings.json`은 프로젝트 루트의 `.claude/settings.json`(프로젝트 전체 적용) 또는 `~/.claude/settings.json`(내 컴퓨터 전체 적용)에 위치합니다.
</Callout>

## 권한 패턴

**도구별 허용/차단** — `Read`, `Write`, `Bash(명령어 패턴)` 형식으로 작성해요.

```
"Bash(git *)"          → git으로 시작하는 모든 명령어 자동 허용
"Bash(npm run test *)" → npm run test로 시작하는 명령어 자동 허용
"Write"                → 모든 파일 쓰기 자동 허용
```

**파일 패턴으로 차단** — 민감한 파일을 읽거나 수정하지 못하게 막아요.

```
"Read(./.env)"              → .env 파일 읽기 차단
"Read(./.env.*)"            → .env로 시작하는 모든 파일 읽기 차단
"Write(./production.config.*)" → 프로덕션 설정 파일 쓰기 차단
```

**스킬 권한** — 특정 스킬의 자동 실행을 허용하거나 차단해요.

```
"Skill(deploy *)"  → deploy로 시작하는 스킬 자동 허용
```

## 설정 범위

권한 설정은 세 가지 수준에서 관리할 수 있으며, 더 좁은 범위의 설정이 우선 적용됩니다.

| 범위 | 파일 위치 | 적용 대상 |
|------|-----------|-----------|
| **조직 정책** | 엔터프라이즈 관리자 설정 | 전체 조직 |
| **개인 설정** | `~/.claude/settings.json` | 내 컴퓨터 전체 |
| **프로젝트 설정** | `.claude/settings.json` | 해당 프로젝트만 |

<Callout type="tip">
  자주 쓰는 명령어(예: git, npm)는 개인 설정(`~/.claude/settings.json`)에, 프로젝트 보안 규칙(예: .env 읽기 차단)은 프로젝트 설정(`.claude/settings.json`)에 넣는 게 좋아요.
</Callout>

---

## 모든 권한 확인 건너뛰기

신뢰할 수 있는 환경(예: CI/CD 자동화, 격리된 개발 환경)에서는 시작 시 플래그를 붙여 권한 확인을 전부 생략할 수 있어요.

```bash
claude --dangerously-skip-permissions
```

<Callout type="warning">
  이 옵션은 Claude가 아무 제한 없이 파일을 수정하고 명령어를 실행할 수 있게 됩니다.\
  본인 컴퓨터의 실제 프로젝트에서는 사용하지 않는 게 좋아요.
</Callout>
