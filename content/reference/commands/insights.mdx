## 언제 쓰나요?

내가 Claude Code를 어떻게 쓰고 있는지 돌아보고, 더 효율적으로 활용하는 방법을 찾고 싶을 때 사용해요.

**이런 상황에서 쓰세요:**

- 어떤 프로젝트에 시간을 많이 쓰고 있는지 확인하고 싶을 때
- Claude에게 반복적으로 같은 지시를 하고 있는 건 아닌지 점검하고 싶을 때
- CLAUDE.md에 넣을 만한 규칙을 자동으로 추출하고 싶을 때

<Callout type="info">
  분석은 내 로컬 머신의 세션 기록만 사용하며, 외부 서버로 데이터를 전송하지 않아요.
</Callout>

---

## 사용법

```
> /insights
```

실행하면 Opus 모델이 지난 30일간의 세션 기록을 분석하고, 리포트를 `~/.claude/usage-data/report.html`에 저장한 뒤 브라우저에서 자동으로 열어줍니다.

각 섹션은 병렬로 생성되기 때문에 전체 리포트가 빠르게 완성돼요.

---

## 리포트에 포함되는 내용

**세션 분석**

| 항목 | 설명 |
|------|------|
| 프로젝트별 활동 | 어떤 프로젝트에 얼마나 시간을 썼는지 |
| 도구 사용 패턴 | 어떤 도구를 자주 쓰는지 |
| 마찰 포인트 | 작업이 자주 막히거나 반복되는 지점 |
| 워크플로우 개선 제안 | 더 효율적으로 쓸 수 있는 방법 |

**CLAUDE.md 규칙 자동 추출**

여러 세션에서 반복적으로 입력한 지시사항을 감지해, CLAUDE.md에 바로 붙여 넣을 수 있는 규칙으로 자동 변환해줍니다. 매번 같은 말을 반복하지 않아도 돼요.

<Callout type="tip">
  리포트가 꽤 솔직한 피드백을 준다고 알려져 있어요.\
  "Claude가 내 코딩 습관을 혹평했다"는 반응이 많을 정도예요.
</Callout>
