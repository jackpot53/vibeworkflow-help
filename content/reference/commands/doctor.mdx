## 언제 쓰나요?

Claude Code가 이상하게 동작하거나, 업데이트가 제대로 됐는지 확인하고 싶을 때 사용해요.

**이런 상황에서 쓰세요:**

- Claude Code가 갑자기 오동작하거나 응답이 이상할 때
- 최신 버전이 설치되어 있는지 확인하고 싶을 때
- 자동 업데이트가 정상적으로 설정되어 있는지 점검할 때
- 설치 경로나 실행 방식을 확인해야 할 때

---

## 사용법

```
> /doctor
```

## 출력 항목 설명

**Diagnostics (현재 실행 정보)**

| 항목 | 설명 |
|------|------|
| `Currently running` | 현재 실행 중인 버전과 설치 방식 |
| `Path` | Claude Code 실행 파일이 있는 경로 |
| `Invoked` | 실제로 호출된 실행 파일 경로 |
| `Config install method` | 설치 방식 (native, npm 등) |
| `Search` | 검색 기능 정상 여부 |

**Updates (업데이트 정보)**

| 항목 | 설명 |
|------|------|
| `Auto-updates` | 자동 업데이트 활성화 여부 |
| `Auto-update channel` | 업데이트 채널 (latest: 최신, stable: 안정) |
| `Stable version` | 현재 안정 릴리즈 버전 |
| `Latest version` | 현재 최신 버전 |

**Version Locks (실행 중인 프로세스)**

현재 실행 중인 Claude Code 프로세스와 PID(프로세스 번호)를 보여줘요. 여러 세션을 동시에 실행 중이라면 여러 항목이 표시될 수 있어요.

<Callout type="tip">
  `Latest version`이 `Currently running`보다 높다면 업데이트가 필요한 상태예요.\
  `/config`에서 자동 업데이트 채널을 확인하거나, 터미널에서 `claude update`를 실행해 수동으로 업데이트할 수 있어요.
</Callout>
