## 언제 쓰나요?

"**by the way(그런데)**"의 줄임말이에요.

Claude가 코드를 작업하는 중에 갑자기 다른 게 궁금해졌을 때, 본 작업 흐름을 끊지 않고 살짝 질문하고 싶을 때 사용해요.
질문과 답변은 대화 기록에 전혀 남지 않기 때문에 Claude의 이후 작업 방향에도 영향을 주지 않아요.

**이런 상황에서 쓰세요:**

- 작업 중 관련 개념이 갑자기 궁금해졌을 때
- 이 세션에서 Claude가 이미 읽은 코드나 결정 사항에 대해 다시 확인하고 싶을 때
- 본 작업과 무관한 짧은 질문이 있는데 흐름을 방해하고 싶지 않을 때

---

## 사용법

```
> /btw [질문 내용]
```

답변은 화면에 **오버레이(팝업)** 형태로 나타나며, 본 대화창과 분리되어 표시됩니다.

```
> /btw 그 설정 파일 이름이 뭐였지?

↳ [btw] .eslintrc.js 파일이에요. 프로젝트 루트에 있어요.
```

답변을 다 읽었으면 **Space**, **Enter**, **Escape** 중 하나를 눌러 닫고 프롬프트로 돌아오세요.

---

## 주요 특징

**Claude가 작업 중일 때도 사용 가능**

Claude가 응답을 처리하는 도중에도 `/btw`를 실행할 수 있어요. 부가 질문은 독립적으로 처리되며, 진행 중인 주 작업을 중단시키지 않아요.

**현재 컨텍스트는 모두 활용**

`/btw`는 이 세션에서 Claude가 이미 읽은 코드, 내린 결정, 대화 내용을 모두 참조할 수 있어요. 단, 새로운 파일을 읽거나 명령어를 실행하거나 검색하는 것은 불가능해요. 이미 알고 있는 것을 확인하는 용도예요.

**후속 질문 없음**

`/btw`는 단 한 번의 답변으로 끝나요. 추가 질문이 필요하다면 일반 메시지를 사용하세요.

**비용이 거의 들지 않음**

부가 질문은 현재 대화의 프롬프트 캐시를 재사용하기 때문에 추가 비용이 최소화돼요.

## 일반 메시지와 뭐가 달라요?

| | `/btw` | 일반 메시지 |
|--|--------|------------|
| 대화 기록에 남는가 | 남지 않음 | 남음 |
| 본 작업에 영향 | 없음 | 있음 |
| 후속 질문 | 불가 | 가능 |
| 파일 읽기·명령어 실행 | 불가 | 가능 |
| 언제 쓰나 | 흐름을 끊지 않고 살짝 물을 때 | 본 작업과 관련된 요청·질문 |

<Callout type="tip">
  `/btw`로 한 질문과 답변은 Claude의 기억에 남지 않아요.\
  본 작업 방향에 영향을 주는 중요한 정보는 일반 메시지로 전달하세요.
</Callout>

---

## `/btw` vs 서브에이전트

둘 다 "본 대화와 분리된" 방식으로 동작하지만, 반대 방향이에요.

| | `/btw` | [서브에이전트](/guides/subagents) |
|--|--------|------------|
| 대화 컨텍스트 | 전체 참조 가능 | 빈 상태로 시작 |
| 도구 사용 | 불가 | 가능 |
| 언제 쓰나 | 이미 알고 있는 것 확인 | 새로운 것을 조사·실행 |

이미 이 세션에서 다룬 내용을 확인하려면 `/btw`를, 새로운 파일을 찾거나 외부 정보를 조사해야 한다면 서브에이전트를 사용하세요.
