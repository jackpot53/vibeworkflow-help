# /btw

<CommandCard
  command="/btw"

---

## 언제 쓰나요?

Claude가 코드를 작업하는 중에 갑자기 딴 게 궁금해졌을 때, 본 작업 흐름을 끊지 않고 살짝 질문하고 싶을 때 사용해요.

"**by the way(그런데)**"의 줄임말이에요.

**이런 상황에서 쓰세요:**

- 로그인 기능 만드는 중에 갑자기 다른 개념이 궁금해졌을 때
- 본 작업과 무관한 짧은 질문이 있을 때
- 지금 흐름을 방해하지 않고 옆에서 살짝 물어보고 싶을 때

---

## 사용법

```
> /btw [질문 내용]
```

Claude는 간단하게 답변하고, 이 질문과 답변은 **본 대화 기록에 남지 않아요**:

```
> /btw REST API랑 GraphQL 차이가 뭐야?

↳ [btw] REST는 URL마다 고정된 데이터를 받아오고, GraphQL은 원하는 필드만
         골라서 한 번에 요청할 수 있어요. 복잡한 데이터를 유연하게 다룰 때
         GraphQL이 유리해요.
```

이후 Claude는 하던 작업을 그대로 이어가요.

---

## 일반 메시지와 뭐가 달라요?

| | `/btw` | 일반 메시지 |
|--|--------|------------|
| 대화 기록에 남는가 | 남지 않음 | 남음 |
| 본 작업에 영향 | 없음 | 있음 |
| 답변 방식 | 짧고 간결하게 | 상황에 따라 다름 |
| 언제 쓰나 | 흐름을 끊지 않고 살짝 물을 때 | 본 작업과 관련된 요청·질문 |

<Callout type="tip">
  `/btw`로 한 질문은 Claude의 기억에 남지 않아요.\
  본 작업에 영향을 주는 중요한 정보는 일반 메시지로 전달하세요.
</Callout>
