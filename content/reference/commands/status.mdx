## 언제 쓰나요?

지금 어떤 모델을 쓰고 있는지, 클로드코드가 제대로 연결돼 있는지 빠르게 확인하고 싶을 때 사용해요.

**이런 상황에서 쓰세요:**

- Claude Code가 이상하게 동작할 때 먼저 상태 확인
- 어떤 모델로 동작하고 있는지 확인하고 싶을 때
- API 연결 문제가 의심될 때
- 버전 확인이 필요할 때

---

## 사용법

```
> /status
```

실행하면 현재 세션 정보가 출력돼요:

```
  Version:          2.1.138
  Session name:     Update sidebar menu labels for vibe-coding-intro plan page
  Session ID:       44c6cefc-7dd9-4fee-a346-ca5a5619d9c7
  cwd:              /Users/amiz/dev/vibeworkflow-help
  Login method:     Claude Max account
  Organization:     john.k@kakaocorp.com's Organization
  Email:            john.k@kakaocorp.com

  Model:            opusplan (claude-sonnet-4-6)
  MCP servers:      1 need auth, 1 disabled · /mcp
  Setting sources:  User settings, Shared project settings, Project local settings
  Esc to cancel
```

---

## 표시 항목

| 항목 | 설명 |
|------|------|
| **Version** | 현재 설치된 클로드코드 버전 |
| **Session name** | 현재 세션의 이름 (대화 내용 기반으로 자동 생성) |
| **Session ID** | 세션 고유 식별자 |
| **cwd** | 클로드코드가 실행 중인 현재 디렉토리 |
| **Login method** | 로그인 방식 (Claude Max account / API key 등) |
| **Organization** | 소속 조직 |
| **Email** | 로그인된 계정 이메일 |
| **Model** | 현재 사용 중인 모델 (플랜명과 실제 모델 ID) |
| **MCP servers** | 연결된 MCP 서버 상태 |
| **Setting sources** | 적용된 설정 파일 출처 |

<Callout type="info">
  토큰 사용량이나 비용 내역은 `/usage`에서 확인하세요.\
  `/status`는 세션 설정과 연결 상태를 빠르게 체크하는 용도예요.
</Callout>

## 세션 상태 점검 명령어 모음

`/status`와 함께 세션 건강 상태를 점검하는 데 유용한 명령어들이에요.

| 명령어 | 역할 |
|--------|------|
| [`/status`](/reference/commands/status) | 버전, 모델, 연결 상태 확인 |
| [`/context`](/reference/commands/context) | 컨텍스트 사용량을 컬러 그리드로 시각화 |
| [`/usage`](/reference/commands/usage) | 요금제 사용 한도 및 속도 제한 근접 여부 확인 |
| [`/doctor`](/reference/commands/doctor) | 설치 환경 및 설정 문제 자동 진단 |

---

## 실전 세션 체크인 루틴

작업이 길어질 때 주기적으로 상태를 점검하는 습관이 좋아요.

```
> /context    # 컨텍스트 얼마나 찼는지 확인
> /usage      # 사용 한도 근접 여부 확인
> /status     # 전반적인 세션 상태 확인
```

컨텍스트가 많이 찼다면 [`/compact`](/reference/commands/compact)로 정리하고, 속도가 느려졌다면 [`/model`](/reference/commands/model)로 더 빠른 모델로 전환하는 식으로 활용할 수 있어요.
