## 언제 쓰나요?

내가 얼마나 썼는지, 한도는 얼마나 남았는지, 비용은 얼마가 나왔는지 종합적으로 파악하고 싶을 때 사용해요.

**이런 상황에서 쓰세요:**

- Claude가 갑자기 응답을 거부하거나 느려질 때 (요청 한도 초과 여부 확인)
- API 키로 사용 중인데 비용이 궁금할 때
- 오늘 얼마나 작업했는지 통계를 보고 싶을 때

---

## 사용법

```
> /usage
```

실행하면 사용량 대시보드가 표시돼요:

```
● Usage Dashboard
  ─── Plan Limits ────────────────────
  Messages today:    42 / 100  (42%)
  ─── Rate Limits ────────────────────
  Requests/min:       3 / 60   (5%)
  ─── Cost ───────────────────────────
  This session:      $0.087
  Today:             $0.243
  ─── Session Stats ──────────────────
  Sessions today:    3
  Avg session time:  24 min
```

<Callout type="info">
  **Claude.ai Pro/Max 구독자**는 요청 횟수 한도가 표시돼요.\
  **API 키 사용자**는 달러 비용이 주로 표시돼요.
</Callout>

---

## 표시 항목

| 항목 | 설명 |
|------|------|
| **Plan Limits** | 하루 사용 가능한 메시지·요청 횟수 대비 현재 사용량 |
| **Rate Limits** | 분당 요청 한도 대비 현재 속도 |
| **Cost** | 현재 세션 및 오늘 하루 예상 비용 (API 키 사용 시) |
| **Session Stats** | 오늘 세션 수, 평균 작업 시간 등 통계 |

## `/context`와 뭐가 달라요?

| | `/usage` | `/context` |
|--|----------|------------|
| 보여주는 것 | 플랜 한도·비용·통계 | 현재 대화의 컨텍스트 창 사용률 |
| 시각화 | 수치 대시보드 | 색상 블록 그래프 |
| 언제 쓰나 | 한도·비용 파악할 때 | 대화 용량 파악할 때 |

## 관련 명령어

| 명령어 | 설명 |
|--------|------|
| `/context` | 현재 대화 컨텍스트 사용률 시각적으로 확인 |
| `/status` | 버전·모델·계정 연결 상태 확인 |
| `/compact` | 대화 요약해서 토큰 절약 |
