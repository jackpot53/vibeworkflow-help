## 언제 쓰나요?

로컬에서 `/plan`으로 하기엔 너무 복잡하거나 시간이 걸리는 계획 작업을, 클라우드에 맡기고 그 사이에 다른 작업을 이어가고 싶을 때 사용해요.

**이런 상황에서 쓰세요:**

- 인증 시스템 전환, 대규모 리팩토링처럼 복잡한 계획이 필요할 때
- 계획이 나오는 동안 터미널을 계속 다른 용도로 쓰고 싶을 때
- 브라우저에서 팀원과 함께 계획을 검토하고 싶을 때

<Callout type="info">
  **사용 조건**
  - Claude Code on the web 계정 (Pro, Max, Team, Enterprise 플랜)
  - GitHub 저장소 연결 필요
</Callout>

---

## 사용법

```
> /ultraplan [작업 설명]
```

실행하면 브라우저가 열리고, 클라우드 세션이 Opus 모델로 계획을 작성합니다.
계획이 완성되면 **브라우저에서 바로 실행**하거나 **터미널로 가져와서 실행**할 수 있어요.

---

## 주요 특징

**클라우드에서 처리**

최대 30분의 클라우드 컴퓨팅 시간을 활용해 복잡한 계획을 작성해요.
로컬 플래닝 모드 대비 약 2배 빠른 처리 속도를 제공합니다.

**브라우저에서 협업**

계획이 작성되는 동안 브라우저에서 실시간으로 검토할 수 있어요.

| 기능 | 설명 |
|------|------|
| 인라인 코멘트 | 계획의 특정 구절에 바로 코멘트 추가 |
| 이모지 리액션 | 섹션별로 승인 또는 수정 요청 |
| 아웃라인 사이드바 | 긴 계획을 섹션 단위로 빠르게 탐색 |

**터미널은 계속 자유롭게**

계획 작업이 클라우드로 넘어가기 때문에, 그 사이에 터미널에서 다른 작업을 계속할 수 있어요.
