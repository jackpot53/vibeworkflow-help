# /model

<CommandCard
  command="/model"

  example="/model claude-opus-4-5"
/>

---

## 언제 쓰나요?

기본 모델보다 더 강력한 모델이 필요할 때, 또는 응답 속도를 높이고 싶을 때 사용해요.\
세션이 끝나면 설정이 초기화되고, 다음 세션에서는 기본 모델로 돌아와요.

<Callout type="info">
  Claude Code의 기본 모델은 **claude-sonnet-4-5**예요.\
  복잡한 설계나 코드 리뷰에는 Opus, 빠른 단순 작업에는 Haiku가 적합해요.
</Callout>

---

## 사용법

```
> /model
```

인자 없이 실행하면 현재 사용 중인 모델을 보여주고, 변경할 모델을 선택할 수 있어요.

모델 ID를 직접 지정할 수도 있어요:

```
> /model claude-opus-4-5
> /model claude-haiku-4-5-20251001
```

---

## 모델 비교

| 모델 | 특징 | 추천 용도 |
|------|------|-----------|
| **claude-opus-4-7** | 가장 강력함 | 복잡한 설계, 아키텍처 리뷰 |
| **claude-sonnet-4-6** | 균형 잡힌 성능 (기본값) | 일반적인 코딩 작업 |
| **claude-haiku-4-5** | 빠르고 가벼움 | 간단한 수정, 빠른 응답 |

<Callout type="tip">
  대부분의 작업은 기본 모델인 **Sonnet**으로 충분해요.\
  긴 코드베이스 분석이나 복잡한 리팩터링에는 **Opus**로 바꿔보세요.
</Callout>
