# /rewind

---

## 언제 쓰나요?

Claude가 여러 단계에 걸쳐 작업하다가 방향이 잘못됐을 때, 특정 시점으로 통째로 되돌리고 싶을 때 사용해요.

**이런 상황에서 쓰세요:**

- 몇 가지 수정을 했는데 결과가 마음에 들지 않아서 여러 단계 전으로 돌아가고 싶을 때
- "저번에 그렇게 했을 때가 더 나았는데" 싶은 시점으로 되돌리고 싶을 때
- `/undo`로는 부족하고 더 많이 되돌려야 할 때

---

## 사용법

```
> /rewind
```

실행하면 저장된 체크포인트 목록이 나타나요:

```
● Available Checkpoints
  1. 10분 전  — 로그인 함수 작성 시작
  2. 23분 전  — API 연결 코드 추가
  3. 41분 전  — 프로젝트 시작
❯ 어느 시점으로 되돌리겠어요?
```

방향키로 원하는 시점을 선택하고 Enter를 누르면 돼요.

---

## 무엇이 되돌아가나요?

`/rewind`는 **대화 내용**과 **파일 변경 사항**을 함께 되돌려요.\
선택한 체크포인트 이후에 일어난 모든 것이 취소돼요.

<Callout type="warning">
  되돌린 이후의 변경 사항은 복구할 수 없어요.\
  되돌리기 전에 중요한 코드가 있다면 따로 복사해두세요.
</Callout>

---

## `/undo`와 뭐가 달라요?

| | `/rewind` | `/undo` |
|--|-----------|---------|
| 되돌리는 범위 | 특정 체크포인트 시점까지 여러 단계 | 바로 직전 파일 변경 1건 |
| 대화 기록 | 함께 되돌아감 | 유지됨 |
| 선택 | 목록에서 골라서 선택 가능 | 선택 없이 즉시 실행 |
| 언제 쓰나 | 여러 단계 전으로 통째로 돌아갈 때 | 방금 수정한 파일 하나를 되돌릴 때 |

<Callout type="tip">
  파일 하나를 잘못 수정했을 때는 가벼운 `/undo`를,\
  작업 방향이 완전히 잘못됐을 때는 `/rewind`를 쓰세요.
</Callout>

---

## 관련 명령어

| 명령어 | 설명 |
|--------|------|
| `/undo` | 바로 직전 파일 변경 사항 되돌리기 |
| `/clear` | 대화 전체 초기화 |
| `/compact` | 대화 요약해서 압축 |
