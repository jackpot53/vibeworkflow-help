## 언제 쓰나요?

바로 코드를 수정하기 전에 Claude가 먼저 코드베이스를 파악하고 계획을 세우도록 할 때 사용해요.\
복잡한 기능 추가, 리팩터링, 버그 원인 파악 등 영향 범위가 넓은 작업에 특히 유용해요.

<Callout type="info">
  Plan 모드에서는 파일 수정, 터미널 명령 실행이 **완전히 차단**돼요.\
  Claude는 읽기와 분석만 할 수 있어요.
</Callout>

---

## 사용법

```
> /plan
```

또는 **Shift + Tab**을 두 번 눌러도 진입할 수 있어요.

---

## 동작 흐름

**1. Plan 모드 진입**

`/plan`을 입력하면 Claude가 읽기 전용 모드로 전환돼요.

**2. 코드베이스 분석 및 질문**

Claude가 관련 파일을 읽고 파악한 뒤, 요구사항이 불명확한 부분은 질문해요.

**3. 구현 계획 생성**

작업 단계, 수정할 파일, 의존 관계 등을 정리한 구체적인 계획을 만들어줘요.

**4. 계획 확인 및 수정**

계획 내용을 직접 수정하거나 피드백을 줄 수 있어요.

**5. Plan 모드 종료 및 실행**

**Shift + Tab**을 다시 눌러 종료하면 Claude가 실행 여부를 확인해요.\
승인하면 계획대로 코드 수정을 시작해요.

<Callout type="tip">
  Plan 모드를 종료할 때 Claude가 실행 전 반드시 확인을 요청해요.\
  의도치 않은 수정이 일어나지 않도록 안전장치가 있어요.
</Callout>

---

## Opus Plan Mode와의 관계

`/model`에서 **Opus Plan Mode**를 선택하면 계획 단계는 Opus(더 강력한 모델)로, 실제 코드 작성은 Sonnet으로 자동 전환돼요.\
복잡한 코드베이스를 깊이 분석해야 할 때 추천해요.

```
> /model
❯ 4. Opus Plan Mode  Use Opus in plan mode, Sonnet otherwise
```
