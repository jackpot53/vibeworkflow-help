# /revise-claude-md

세션을 마무리할 때, 이번 대화를 통해 새로 발견한 내용·수정한 내용·주의사항 등을 CLAUDE.md에 반영해주는 명령어예요.

---

## 언제 쓰나요?

세션을 통해 새롭게 알게 된 것들이 다음 번에도 기억되길 바랄 때 사용해요.

**이런 상황에서 쓰세요:**

- 작업 중에 "이건 CLAUDE.md에 적어두면 좋겠다"는 생각이 들었을 때
- 오늘 세션에서 새로운 규칙·패턴·주의사항을 발견했을 때
- 매 세션마다 같은 설명을 반복하고 있다는 느낌이 들 때
- 세션을 마무리하기 전 학습 내용을 정리하고 싶을 때

---

## 사용법

```
> /revise-claude-md
```

실행하면 Claude가 이번 세션의 대화를 분석해서 CLAUDE.md에 추가할 내용을 제안해요.

---

## 실행 흐름

**1단계 — 세션 분석**

Claude가 이번 대화에서 새로 알게 된 것들을 정리해요:

```
이번 세션에서 발견된 내용을 분석하고 있어요...

- 새로 추가된 환경변수: REDIS_URL
- 기존과 다른 테스트 실행 방법 확인
- API 응답 형식 변경 사항 발견
```

**2단계 — 추가 내용 제안**

CLAUDE.md에 추가할 내용을 diff 형태로 보여줘요:

```
### Update: ./CLAUDE.md

이유: 이번 세션에서 Redis 설정이 필요하다는 것을 발견했어요.

+ ## 환경변수
+
+ - `DATABASE_URL` — PostgreSQL 연결 문자열
+ - `REDIS_URL`    — Redis 캐시 서버 (세션 관리용, 필수)
```

**3단계 — 승인 후 업데이트**

내용을 확인하고 승인하면 CLAUDE.md에 바로 반영돼요.

<Callout type="tip">
  세션을 끝내기 전 습관적으로 `/revise-claude-md`를 실행하면\
  CLAUDE.md가 점점 더 풍부해지고, 다음 세션부터 Claude가 훨씬 빠르게 파악해요.
</Callout>

---

## `#` 단축키

세션 도중 CLAUDE.md에 기록하고 싶은 게 생겼을 때는\
`#` 키를 눌러 빠르게 추가할 수 있어요:

```
# 이 프로젝트는 pnpm을 써야 해, npm 쓰면 lock 파일 충돌 남
```

Claude가 이 내용을 CLAUDE.md에 바로 반영해요.
