# claude-md-improver

CLAUDE.md 파일들을 현재 코드베이스와 비교해 품질을 감사하고, 승인 후 자동으로 개선해주는 스킬이에요.

<Callout type="info">
  이 스킬은 **CLAUDE.md 파일을 직접 수정**할 수 있어요.\
  변경 전에 항상 품질 리포트를 먼저 보여주고 승인을 구해요.
</Callout>

---

## 언제 쓰나요?

- 코드가 많이 바뀌었는데 CLAUDE.md는 오래된 것 같을 때
- Claude가 자꾸 잘못된 명령어나 파일 구조를 언급할 때
- 새 팀원이 합류해서 CLAUDE.md를 점검하고 싶을 때
- 주기적인 CLAUDE.md 관리를 하고 싶을 때

---

## 사용법

스킬은 슬래시 명령어 없이 자연어로 호출해요:

```
> CLAUDE.md 감사해줘
> CLAUDE.md 파일 확인해줘
> 프로젝트 기억 최적화해줘
> CLAUDE.md가 최신 상태인지 체크해줘
```

---

## 실행 흐름

**1단계 — 파일 탐색**

프로젝트 전체에서 CLAUDE.md 파일을 모두 찾아요:

```
./CLAUDE.md
./packages/api/CLAUDE.md
~/.claude/CLAUDE.md
```

**2단계 — 품질 평가**

각 파일을 아래 기준으로 점수를 매겨요:

| 평가 항목 | 중요도 | 내용 |
|-----------|--------|------|
| 명령어·워크플로우 | 높음 | 빌드·테스트·배포 명령어가 있는가? |
| 구조 명확성 | 높음 | Claude가 코드베이스를 이해할 수 있는가? |
| 비직관적 패턴 | 중간 | 주의사항·특이점이 기록돼 있는가? |
| 간결성 | 중간 | 불필요한 설명 없이 간결한가? |
| 최신성 | 높음 | 현재 코드 상태를 반영하는가? |
| 실행 가능성 | 높음 | 명령어가 바로 복붙해서 쓸 수 있는가? |

**3단계 — 품질 리포트 출력**

수정 전에 항상 이런 리포트를 먼저 보여줘요:

```
## CLAUDE.md Quality Report

### 요약
- 파일 수: 2개
- 평균 점수: 61/100
- 업데이트 필요: 1개

### ./CLAUDE.md — 61/100 (등급: C)

| 항목 | 점수 | 비고 |
|------|------|------|
| 명령어·워크플로우 | 12/20 | 테스트 명령어 누락 |
| 구조 명확성 | 14/20 | |
| 비직관적 패턴 | 8/15 | 환경변수 설정 누락 |
| 간결성 | 12/15 | |
| 최신성 | 8/15 | 빌드 명령어가 오래됨 |
| 실행 가능성 | 7/15 | |

문제점:
- npm run test 명령어가 없음
- DATABASE_URL 환경변수 설정 방법 없음
- 오래된 빌드 명령어 (npm build → npm run build)
```

**4단계 — 승인 후 업데이트**

리포트를 확인하고 승인하면 변경 사항을 적용해요:

```
### Update: ./CLAUDE.md

이유: 테스트 명령어가 없어서 매 세션마다 혼란이 생겼음

+ ## 테스트
+
+ ```bash
+ npm run test        # 전체 테스트
+ npm run test:watch  # 변경 감지 테스트
+ ```
```

## 품질 등급 기준

| 등급 | 점수 | 상태 |
|------|------|------|
| **A** | 90~100 | 완벽 — 포괄적이고 최신 상태 |
| **B** | 70~89 | 양호 — 소소한 빈틈 있음 |
| **C** | 50~69 | 보통 — 핵심 섹션 일부 누락 |
| **D** | 30~49 | 미흡 — 내용이 너무 적거나 오래됨 |
| **F** | 0~29 | 없거나 심각하게 낡음 |

---

## 자주 발견되는 문제

- 더 이상 작동 안 하는 명령어
- 변경된 파일 구조
- 없어진 환경변수 설명
- 바뀐 테스트 명령어
- 새로 생긴 특이점이나 주의사항 미기록
