# CLAUDE.md Management

CLAUDE.md 파일을 관리하는 두 가지 도구를 제공하는 공식 플러그인이에요.\
코드베이스가 바뀌어도 CLAUDE.md가 항상 최신 상태를 유지하도록 도와줘요.

---

## 왜 필요한가요?

CLAUDE.md는 Claude Code가 프로젝트를 시작할 때 자동으로 읽는 파일이에요.\
여기에 프로젝트 정보가 잘 정리돼 있을수록 Claude가 더 정확하게 도와줄 수 있어요.

하지만 시간이 지나면서 코드가 바뀌면 CLAUDE.md 내용이 낡아지고, 매 세션마다 같은 내용을 설명하게 되는 문제가 생겨요.

이 플러그인은 그 관리 부담을 자동화해줘요.

---

## 두 가지 도구

| | [claude-md-improver](./claude-md-improver) | [/revise-claude-md](./revise-claude-md) |
|---|---|---|
| **종류** | 스킬 (대화로 호출) | 슬래시 명령어 |
| **하는 일** | CLAUDE.md 품질 감사 및 개선 | 이번 세션 학습 내용 기록 |
| **쓸 때** | 코드베이스가 많이 바뀌었을 때 | 세션 끝에 배운 것들을 정리할 때 |
| **호출 방법** | "CLAUDE.md 감사해줘" | `/revise-claude-md` |

<Callout type="tip">
  두 도구는 서로 보완적이에요.\
  **claude-md-improver**는 코드 기준으로 CLAUDE.md를 점검하고,\
  **/revise-claude-md**는 대화 중 새로 알게 된 내용을 추가해요.
</Callout>

---

## 플러그인 설치

Claude Code에서 플러그인을 설치해요:

```
> /plugins add claude-md-management
```

설치 후 두 도구가 바로 사용 가능해요.

---

## CLAUDE.md 파일 위치

이 플러그인은 프로젝트의 모든 CLAUDE.md 파일을 자동으로 찾아줘요:

| 위치 | 파일 | 용도 |
|------|------|------|
| 프로젝트 루트 | `./CLAUDE.md` | 팀 전체가 공유하는 주요 맥락 |
| 개인 로컬 | `./.claude.local.md` | 나만의 개인 설정 (공유 안 함) |
| 전체 기본값 | `~/.claude/CLAUDE.md` | 모든 프로젝트에 적용되는 사용자 설정 |
| 패키지별 | `./packages/*/CLAUDE.md` | 모노레포에서 각 패키지 맥락 |
