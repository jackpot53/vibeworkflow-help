# 배포하기

코드가 완성됐다면 이제 세상에 공개할 차례예요.\
**Vercel**은 가장 쉽고 빠르게 웹 서비스를 배포할 수 있는 플랫폼입니다.\
GitHub에 코드만 올리면 클릭 몇 번으로 끝납니다.

<Callout type="tip">
  Vercel은 개인 프로젝트 기준 무료로 사용할 수 있어요. 신용카드 없이 가입 가능합니다.
</Callout>

---

## 시작 전 확인사항

배포를 시작하기 전에 아래 두 가지가 준비되어 있어야 해요.

| 필요한 것 | 설명 |
|-----------|------|
| **GitHub 계정** | 코드를 올려두는 저장소. 없다면 [github.com](https://github.com)에서 무료 가입 |
| **Vercel 계정** | 배포 플랫폼. GitHub 계정으로 바로 가입 가능 |

<Callout type="info">
  아직 코드가 GitHub에 올라가 있지 않다면, 먼저 Claude Code에게 "GitHub에 코드 올려줘"라고 요청하세요.
</Callout>

---

## Vercel 배포 단계별 가이드

### <StepBadge step={1} /> Vercel 가입하기

1. [vercel.com](https://vercel.com)에 접속해요.
2. 우측 상단 **Sign Up** 버튼을 클릭해요.
3. **Continue with GitHub**를 선택하세요.

   > GitHub 계정으로 로그인하면 내 저장소(repository)를 Vercel이 바로 읽어올 수 있어요.

4. GitHub 권한 요청 화면이 나오면 **Authorize Vercel**을 클릭하세요.

<Callout type="tip">
  가입 과정에서 팀 생성 화면이 나오면 **Skip**을 눌러 넘어가도 됩니다. 혼자 사용할 땐 팀이 필요 없어요.
</Callout>

---

### <StepBadge step={2} /> 새 프로젝트 만들기

1. 로그인 후 대시보드에서 **Add New → Project** 버튼을 클릭해요.
2. **Import Git Repository** 목록에서 내 GitHub 저장소가 보여요.
3. 배포할 저장소 오른쪽의 **Import** 버튼을 클릭하세요.

<Screenshot
  src="https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/import-git-repo.png"
  alt="Vercel에서 GitHub 저장소 목록과 Import 버튼"
  caption="내 GitHub 저장소 목록이 표시되고, Import 버튼을 누르면 됩니다"
/>

<Callout type="info">
  저장소가 목록에 안 보인다면 **Adjust GitHub App Permissions** 링크를 클릭해서 Vercel이 해당 저장소에 접근할 수 있도록 허용해 주세요.
</Callout>

---

### <StepBadge step={3} /> 프로젝트 설정하기

Import 후 설정 화면이 나와요. 대부분 자동으로 잡혀 있지만 확인해볼 것들이 있어요.

<Screenshot
  src="https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/configure-project.png"
  alt="Vercel 프로젝트 설정 화면 — Project Name, Framework Preset, 환경 변수"
  caption="프로젝트 이름, 프레임워크, 환경 변수를 설정하는 화면입니다"
/>

#### Framework Preset

Vercel이 프로젝트 종류를 자동으로 감지해요.

| 내 프로젝트 | 자동 감지 값 |
|-------------|-------------|
| Next.js | **Next.js** |
| React | **Create React App** |
| 기타 | **Other** |

자동으로 잡힌 값이 맞다면 그대로 두세요.

#### 환경 변수 (Environment Variables)

`.env` 파일에 API 키나 비밀 값이 있다면 여기에 입력해야 해요.\
로컬에서만 쓰던 값들이 배포 환경에서도 동작하려면 반드시 등록해야 합니다.

<Screenshot
  src="https://7nyt0uhk7sse4zvn.public.blob.vercel-storage.com/docs-assets/static/docs/concepts/projects/environment-variables/env-var-section-light.png"
  alt="Vercel 환경 변수 입력 화면 — Key, Value 입력 필드"
  caption="KEY와 VALUE를 입력하고 Add를 누르면 환경 변수가 등록됩니다"
/>

1. **Environment Variables** 항목을 펼쳐요.
2. **NAME**에 변수명 (예: `NEXT_PUBLIC_API_KEY`), **VALUE**에 값을 입력해요.
3. 변수가 여러 개면 **Add another**로 계속 추가하세요.

<Callout type="warning">
  `.env` 파일은 절대 GitHub에 올리면 안 돼요. 비밀 키가 외부에 노출됩니다. Vercel 환경 변수 설정 화면에서만 입력하세요.
</Callout>

---

### <StepBadge step={4} /> 배포 시작

설정이 끝나면 화면 하단의 **Deploy** 버튼을 클릭하세요.

배포 진행 화면이 나타나며 로그가 실시간으로 흘러요.\
보통 **1~3분** 안에 완료됩니다.

배포가 성공하면 🎉 축하 화면과 함께 **내 서비스 URL**이 나타나요.

<Screenshot
  src="https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/deployed-project.png"
  alt="Vercel 배포 완료 대시보드 — Production Deployment, Status: Ready"
  caption="배포가 완료되면 .vercel.app 주소로 서비스에 접근할 수 있어요"
/>

```
https://내-프로젝트명.vercel.app
```

이 링크를 누구에게나 공유할 수 있어요!

<Callout type="tip">
  배포 중 오류가 났다면 로그를 복사해서 Claude Code에게 붙여넣어 보세요. "이 배포 오류 고쳐줘"라고 하면 원인을 찾아줍니다.
</Callout>

## 배포 후: 자동 배포 설정

Vercel과 GitHub가 연결되면 이후엔 자동으로 배포돼요.

```
코드 수정 → GitHub에 push → Vercel이 자동 감지 → 자동 배포
```

Claude Code가 코드를 수정하고 GitHub에 올리는 순간, 새 버전이 자동으로 반영됩니다.\
따로 배포 버튼을 누를 필요가 없어요.

<Callout type="info">
  **프리뷰 배포**: `main` 브랜치 외의 다른 브랜치에 push하면 별도의 미리보기 URL이 생성돼요. 실제 서비스에 영향 없이 테스트할 수 있어요.
</Callout>

## 커스텀 도메인 연결하기 (선택)

`내도메인.com` 같은 주소를 연결하고 싶다면 도메인을 구매한 뒤 Vercel에서 설정할 수 있어요.

1. Vercel 프로젝트 대시보드에서 **Settings → Domains**로 이동해요.
2. 원하는 도메인 주소를 입력하고 **Add**를 클릭해요.

<Screenshot
  src="https://7nyt0uhk7sse4zvn.public.blob.vercel-storage.com/docs-assets/static/docs/domains/enter-domain-input-light.png"
  alt="Vercel 도메인 입력 화면"
  caption="연결할 도메인 주소를 입력하는 화면입니다"
/>

3. 도메인 구매처(가비아, Cloudflare 등)에서 Vercel이 알려주는 DNS 값을 설정해요.

<Screenshot
  src="https://7nyt0uhk7sse4zvn.public.blob.vercel-storage.com/docs-assets/static/docs/concepts/projects/custom-domains/new-domain-apex-light.png"
  alt="Vercel 도메인 DNS A 레코드 설정 화면"
  caption="Vercel이 알려주는 DNS 값을 도메인 구매처에서 동일하게 입력하면 됩니다"
/>

<Callout type="tip">
  도메인 DNS 설정은 반영까지 최대 48시간이 걸릴 수 있어요. 보통은 몇 분 안에 됩니다.
</Callout>

## 자주 하는 질문

**Q. 배포하면 비용이 드나요?**\
개인 프로젝트는 Vercel 무료 플랜으로 충분합니다. 월 100GB 대역폭, 무제한 배포 횟수를 제공해요.

**Q. 배포한 걸 내리고 싶어요.**\
Vercel 대시보드에서 프로젝트를 선택하고 **Settings → Delete Project**로 삭제할 수 있어요.

**Q. 배포는 됐는데 화면이 이상해요.**\
로컬에서 잘 되던 것이 배포 후 깨진다면 환경 변수 누락이 가장 흔한 원인이에요. Vercel 환경 변수 설정을 다시 확인해보세요.
