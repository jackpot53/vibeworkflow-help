# 04. 배포하기

코드가 GitHub에 올라갔다면 이제 세상에 공개할 차례예요.\
**Vercel**을 이용하면 클릭 몇 번으로 내 사이트가 실제 URL을 갖게 돼요.

<Callout type="tip">
Vercel은 개인 프로젝트 기준 무료로 사용할 수 있어요. 신용카드 없이 GitHub 계정으로 바로 가입할 수 있어요.
</Callout>

---

## 시작 전 확인사항

배포 전에 두 가지가 준비되어 있어야 해요.

| 필요한 것 | 확인 방법 |
|-----------|----------|
| **코드가 GitHub에 올라가 있어야 해요** | 이전 단계에서 `git push`까지 완료했다면 OK |
| **Vercel 계정이 있어야 해요** | [vercel.com](https://vercel.com)에서 GitHub 계정으로 무료 가입 |

---

## 배포 순서

<Steps>

### Vercel에 가입해요

1. [vercel.com](https://vercel.com)에 접속해요.
2. 우측 상단 **Sign Up** 버튼을 클릭해요.
3. **Continue with GitHub**를 선택하세요.
4. GitHub 권한 요청 화면에서 **Authorize Vercel**을 클릭하세요.

<Callout type="tip">
  가입 중 팀 생성 화면이 나오면 **Skip**을 눌러도 돼요. 혼자 쓸 땐 팀이 필요 없어요.
</Callout>

### 프로젝트를 가져와요

1. 로그인 후 대시보드에서 **Add New → Project**를 클릭해요.
2. **Import Git Repository** 목록에서 내 저장소를 찾아요.
3. 저장소 오른쪽 **Import** 버튼을 클릭하세요.

<Callout type="info">
  저장소가 목록에 보이지 않는다면 **Adjust GitHub App Permissions** 링크를 클릭해서 Vercel이 해당 저장소에 접근하도록 허용해 주세요.
</Callout>

### 설정을 확인해요

**Framework Preset**은 Vercel이 자동으로 감지해줘요. Next.js로 잡혀 있다면 그대로 두세요.

환경 변수(`.env`)가 있다면 **Environment Variables** 항목을 펼쳐서 입력해야 해요.

<Callout type="warning">
  `.env` 파일은 절대 GitHub에 올리면 안 돼요. Vercel 설정 화면에서만 입력하세요.
</Callout>

### Deploy를 눌러요

화면 하단의 **Deploy** 버튼을 클릭하세요.\
보통 **1~3분** 안에 완료되고, 배포가 성공하면 내 사이트 URL이 나타나요.

```
https://내-프로젝트명.vercel.app
```

이 링크를 바로 공유할 수 있어요!

<Callout type="tip">
  배포 중 오류가 났다면 로그를 복사해서 Claude Code에게 붙여넣어 보세요.\
  **"이 배포 오류 고쳐줘"** 라고 하면 원인을 찾아줘요.
</Callout>

</Steps>

---

## 이후엔 자동으로 배포돼요

Vercel과 GitHub가 연결되면 다음부터는 push만 해도 자동으로 반영돼요.

```
코드 수정 → GitHub에 push → Vercel이 자동 감지 → 자동 배포
```

Claude Code가 코드를 수정하고 GitHub에 올리는 순간, 새 버전이 자동으로 배포됩니다.\
따로 배포 버튼을 누를 필요가 없어요.

<Callout type="info">
  `main` 브랜치 외의 다른 브랜치에 push하면 별도의 미리보기 URL이 생성돼요.\
  실제 서비스에 영향 없이 테스트할 수 있어요.
</Callout>

---

## 배포가 완료됐어요

포트폴리오 사이트가 인터넷에 공개됐어요. 이제부터는 GitHub에 push할 때마다 자동으로 최신 버전이 반영돼요.

더 자세한 배포 설정이 궁금하다면 [배포하기 가이드](/guides/deployment)를 참고하세요.
