# 08. API 연동

포트폴리오에 실제 데이터를 연결하면 훨씬 살아있는 사이트가 돼요.\
취업공고 사이트 API를 연동해서 실시간 채용정보를 보여주는 기능을 만들어볼게요.

---

## 사람인 채용정보 API

개인 포트폴리오에 취업공고를 연동할 때 가장 많이 쓰이는 선택지예요.

| 항목 | 내용 |
|------|------|
| **엔드포인트** | `oapi.saramin.co.kr` |
| **응답 형식** | JSON |
| **제공 정보** | 회사명, 포지션, 마감일, 직무 분류 등 |
| **이용 방법** | 신청 및 승인 후 API 키 발급 |

API 키는 [사람인 채용정보 API 가이드](https://oapi.saramin.co.kr/guide/job-search)에서 신청할 수 있어요.

<Callout type="info">
  API 키 발급에 1~2일 정도 소요될 수 있어요. 미리 신청해두는 게 좋아요.\
  API 이용신청을 했는데 메일로 아직 답변이 없네요 ^^ 
</Callout>

    API 상세 페이지 왼쪽 패널에서 엔드포인트를 선택하면 오른쪽에 테스트 UI가 나타나요.

    2. 버튼을 클릭해요

    3. 우측 하단 응답 패널에서 실제 JSON 응답 확인

    1. **Subscribe** 버튼 클릭 → 무료 플랜(basic plan) 선택
    2. API 키(`X-RapidAPI-Key`) 확인

    <Callout type="info">
      RapidAPI의 API 키는 서비스마다 다른 게 아니에요. 내 계정에 연결된 키 **하나**로 구독한 모든 API를 호출해요.
    </Callout>
  </Tabs.Tab>
  <Tabs.Tab>
    ### API 프로토콜 확인

    **Code Snippets** 탭을 열면 언어별 호출 코드가 자동 생성돼요.\
    여기서 확인해야 할 것:

    | 항목 | 위치 |
    |------|------|
    | **엔드포인트 URL** | `url` 값 |
    | **필수 헤더** | `X-RapidAPI-Key`, `X-RapidAPI-Host` |
    | **요청 파라미터** | `querystring` 또는 `params` 객체 |
    | **응답 구조** | 플레이그라운드 응답 패널 참고 |

    좌측 패널에서 **API Overview**를 클릭하면 엔드포인트·헤더·파라미터 등 API 상세 프로토콜을 한눈에 확인할 수 있어요.

    <Screenshot

  </Tabs.Tab>

## API를 클로드코드로 구현하는 가장 현명한 방법

API를 클로드코드로 구현할 때 가장 흔한 실수는 **"사람인 API 연동해줘"라고만 하는 거예요.**

Claude가 API 스펙을 모르는 상태에서 코드를 짜면 엔드포인트, 파라미터 이름, 응답 구조를 틀리게 추측해요. 그러면 틀린 코드를 고치는 데 시간을 더 쓰게 돼요.

<Callout type="tip">
  <span className="inline-block text-xs font-semibold tracking-widest uppercase px-2 py-0.5 mb-2 bg-emerald-500/15 text-emerald-600 dark:text-emerald-400 border border-emerald-500/30">Best Practice</span>
  클로드코드에게 API를 구현하도록 시키기 **전에**, API 공식 문서를 `docs/api/` 폴더에 `spec.md` 파일로 저장해두세요.\
  DB의 ERD 파일처럼, Claude가 스펙 파일을 참고하면 추측 없이 정확한 코드를 짜줘요.
</Callout>

## spec.md 파일 만들기

### 1단계 — 클로드코드에게 스펙 파일을 만들어달라고 해요

두 가지 방법 중 편한 걸 쓰면 돼요.

**방법 A — API 문서 URL을 넘겨주는 경우**

[사람인 API 가이드 페이지](https://oapi.saramin.co.kr/guide/job-search)를 열고, 아래처럼 입력해요.

<VibeBlock>
```
https://oapi.saramin.co.kr/guide/job-search

이 페이지에 사람인 채용정보 API의 request/response 스펙이 있어.
이 내용을 바탕으로 docs/api/saramin-spec.md 파일을 만들어줘.
엔드포인트, 요청 파라미터, 응답 필드, 예시를 모두 포함해줘.
```
</VibeBlock>

**방법 B — API 스펙을 직접 붙여넣는 경우**

URL 접근이 안 되거나 내부 API인 경우, 스펙 내용을 직접 붙여넣어도 돼요.

<VibeBlock>
```
이게 우리가 쓸 API 스펙이야. 이걸 바탕으로 docs/api/saramin-spec.md 파일 만들어줘.

## Request
GET https://oapi.saramin.co.kr/job
Headers:
  access-key: {API_KEY}

Query Parameters:
  keywords  string  검색 키워드
  loc_mcd   string  지역 코드 (예: 101000 서울)
  count     number  결과 수 (기본 10, 최대 110)
  start     number  시작 페이지 (기본 0)

## Response

  "jobs": {
    "count": 10,
    "start": 0,
    "total": "1234",
    "job": [
      {
        "id": "12345",

          "detail": { "name": "카카오" }
        },
        "position": {
          "title": "프론트엔드 개발자",
          "location": { "name": "서울" },
          "job-type": { "name": "정규직" },
          "experience-level": { "name": "경력 3년↑" }

## 요청 파라미터

| count | number | N | 결과 수 (기본 10, 최대 110) |

{
  "jobs": {
    "count": 100,
    "start": 0,
    "total": "1000",
    "job": [
      {
        "id": "12345",

스펙 파일이 준비됐으면 클로드코드에게 구현을 맡겨요.

### 기본 연동

<VibeBlock>
```
docs/api/saramin-spec.md 파일을 참고해서
사람인 API로 채용공고 목록을 가져오는 기능을 만들어줘.
키워드로 검색할 수 있고, 결과를 카드 형태로 보여줘.
API 키는 환경변수 SARAMIN_API_KEY로 관리해줘.

</VibeBlock>

## 다른 API에도 똑같이 적용해요

이 방법은 사람인 API에만 쓰는 게 아니에요. 어떤 API든 같은 순서로 하면 돼요.

1. API 공식 문서 링크를 클로드코드에 붙여넣어요
2. `docs/api/` 폴더에 스펙 파일을 만들어달라고 해요
3. CLAUDE.md에 스펙 파일 참고하라고 지침을 추가해요
4. 구현을 맡겨요

공공데이터 포털, 카카오맵, 날씨 API, 유튜브 API 등 어떤 API든 이 방식으로 클로드코드와 함께 빠르게 연동할 수 있어요.
