## Plan 모드 전에 먼저 모델을 설정해요

클로드코드는 작업의 복잡도에 따라 사용하는 AI 모델을 바꿀 수 있어요.\
간단한 수정엔 빠르고 저렴한 모델을, 복잡한 계획이 필요할 땐 더 똑똑한 모델을 선택하는 식이에요.\
작업에 맞는 모델을 쓰면 속도와 비용을 모두 잡을 수 있어요.

선택할 수 있는 모델은 세 가지예요.

| 모델 | 속도 | 비용 | 특징 |
|------|------|------|------|
| **Haiku** | 가장 빠름 | 가장 저렴 | 간단한 수정, 짧은 질문에 적합해요 |
| **Sonnet** | 빠름 | 중간 | 일반적인 코딩 작업에 가장 균형 잡힌 선택이에요. 기본값이에요 |
| **Opus** | 느림 | 가장 비쌈 | 가장 똑똑해요. 복잡한 구조 분석이나 계획 수립에 강해요 |

**1단계 — 클로드코드 터미널에서 [`/model`](/reference/commands/model)을 입력해요**

<SlashBlock>
```
/model
```
</SlashBlock>

**2단계 — 메뉴에서 `Opus Plan Mode`를 선택해요**

방향키로 4번 항목으로 이동한 뒤 Enter를 눌러요.

<ClaudeBlock>
```
  1. Default (recommended)  Opus 4.7 with 1M context
  2. Sonnet                 Sonnet 4.6
  3. Haiku                  Haiku 4.5
❯ 4. Opus Plan Mode ✔       Use Opus in plan mode, Sonnet otherwise
```
</ClaudeBlock>

4번 항목인 **Opus Plan Mode**는 조금 특별해요.\
계획을 세울 때는 가장 똑똑한 Opus가 투입되고, 실제 코드를 짤 때는 빠른 Sonnet으로 자동으로 바뀌어요.\
"머리 쓸 때는 Opus, 손발은 Sonnet"인 셈이에요.

<Callout type="info">
  <span className="inline-block text-xs font-semibold tracking-widest uppercase px-2 py-0.5 mb-2 bg-emerald-500/15 text-emerald-600 dark:text-emerald-400 border border-emerald-500/30">Best Practice</span>
  기본 모델(Sonnet)로도 Plan 모드를 쓸 수 있지만,\
  **Opus Plan Mode**로 설정하면 계획 단계에서만 Opus가 투입돼 더 깊이 분석해줘요.\
  코드 작성 단계는 다시 Sonnet으로 자동 전환되니 비용도 효율적이에요.
</Callout>

## Plan 모드로 전환

Plan 모드로 전환하기 위해 클로드코드 CLI에서 `/plan`을 입력하거나, **Shift + Tab**을 두 번 눌러도 돼요.

<SlashBlock>
```
/plan
```
</SlashBlock>

클로드코드 익스텐션(안티그래비티)에서는 입력창 오른쪽 아래의 모드 버튼을 클릭한 뒤 **Plan mode**를 선택하면 돼요.

<Screenshot src="/images/planmode.png" alt="클로드코드 익스텐션 Plan mode 선택 화면" />

이 명령어를 입력하는 순간 Claude가 Plan 모드로 전환돼요.\
이 상태에서 원하는 걸 말하면 코드를 바로 짜는 대신, 어떻게 만들지 계획부터 세워줘요.\
파일을 건드리지 않고 분석과 계획만 하는 안전한 상태예요.

| 단계 | 사용 모델 | 역할 |
|------|-----------|------|
| `/plan` 모드 | **Opus** | 코드베이스 분석, 구조 파악, 계획 수립 |
| 코드 작성 | **Sonnet** | 빠르고 안정적인 구현 |

자세한 내용은 [`/model` 레퍼런스](/reference/commands/model)와 [`/plan` 레퍼런스](/reference/commands/plan)를 참고하세요.

## 이번 실습 기술 스택

이번 실습에서 사용할 기술 스택은 아래와 같아요. Claude에게 말을 걸 때 이 스택을 함께 알려주면 더 정확하게 세팅해줘요.

| 기술 | 한 줄 설명 |
|------|-----------|
| [**Next.js**](https://nextjs.org) | 웹사이트를 만드는 React 기반 프레임워크예요. 폴더 구조만으로 페이지가 자동으로 만들어져요. |
| [**TypeScript**](https://www.typescriptlang.org) | JavaScript에 타입을 추가한 언어예요. 코드 실수를 미리 잡아줘서 더 안전한 개발이 가능해요. |
| [**App Router**](https://nextjs.org/docs/app) | Next.js의 최신 라우팅 방식이에요. `app/` 폴더 안에 파일을 만들면 자동으로 URL이 생성돼요. |
| [**Supabase**](https://supabase.com) | 데이터베이스 + 로그인 기능을 쉽게 쓸 수 있는 서비스예요. 백엔드 없이도 데이터를 저장할 수 있어요. |
| [**Vercel**](https://vercel.com) | Next.js를 만든 회사가 운영하는 배포 플랫폼이에요. GitHub에 올리면 자동으로 배포돼요. |
| [**Tailwind CSS**](https://tailwindcss.com) | 클래스 이름으로 디자인하는 CSS 도구예요. `text-lg`, `bg-blue-500` 같은 이름으로 스타일을 바로 적용해요. |
| [**shadcn/ui**](https://ui.shadcn.com) | 버튼, 카드, 모달 같은 UI 컴포넌트 모음이에요. 복사해서 바로 쓸 수 있고 커스터마이징도 쉬워요. |
