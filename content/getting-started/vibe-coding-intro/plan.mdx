# 03. 계획(PLAN)하기

## 원래는 이런 것들을 해야 해요

본격적인 개발 전에는 보통 이런 과정을 거쳐요:

- **브레인스토밍** — 어떤 제품을 만들지, 핵심 가치는 무엇인지
- **기능 정의** — 어떤 기능을 넣을지, 우선순위는 어떻게 할지
- **경쟁 제품 리서치** — 비슷한 제품들의 특징, 강점, 차별화 포인트 분석
- **사용자 흐름 설계** — 사용자가 어떤 경로로 움직이는지
- **기술 스택 결정** — 어떤 언어, 프레임워크, 인프라를 쓸지

<Callout type="info">
  실제 서비스를 만들 때는 이 과정이 매우 중요해요.\
  하지만 이번 실습은 **심플한 포트폴리오 사이트**를 만드는 것이 목표라 이 과정은 건너뛸게요.
</Callout>

---

## 대신 Claude Code의 Plan 모드를 써볼게요

본격적인 기획 없이도 **Claude Code의 [`/plan` 모드](/reference/commands/plan)** 로 비슷한 효과를 낼 수 있어요.

Plan 모드에서 Claude는 파일을 **절대 수정하지 않아요**.\
코드를 읽고 분석하는 것만 할 수 있어요.\
요구사항이 불명확하면 먼저 질문을 던지고, 충분히 파악한 뒤 어떤 파일을 어떤 순서로 수정할지 단계별 계획을 만들어줘요.\
계획을 확인받은 후에만 실제 코드 작성을 시작하기 때문에 의도치 않은 수정이 일어나지 않아요.

실수 없이 큰 그림을 먼저 잡을 수 있는 안전한 첫 단계예요.

## Plan 모드 전에 먼저 모델을 설정해요

Plan 모드를 시작하기 **전에** 모델을 **Opus Plan Mode**로 바꿔두는 게 좋아요.

<Callout type="info">
  <span className="inline-block text-xs font-semibold tracking-widest uppercase px-2 py-0.5 mb-2 bg-emerald-500/15 text-emerald-600 dark:text-emerald-400 border border-emerald-500/30">Best Practice</span>
  기본 모델(Sonnet)로도 Plan 모드를 쓸 수 있지만,\
  **Opus Plan Mode**로 설정하면 계획 단계에서 Claude의 가장 강력한 모델인 Opus가 투입돼 더 깊이 분석해줘요.\
  코드 작성 단계는 다시 Sonnet으로 자동 전환되니 비용도 효율적이에요.
</Callout>

**1단계 — Claude Code 터미널에서 `/model`을 입력해요**

```
> /model
```

**2단계 — 메뉴에서 `Opus Plan Mode`를 선택해요**

방향키로 4번 항목으로 이동한 뒤 Enter를 눌러요.

```
  1. Default (recommended)  Opus 4.7 with 1M context
  2. Sonnet                 Sonnet 4.6
  3. Haiku                  Haiku 4.5
❯ 4. Opus Plan Mode ✔       Use Opus in plan mode, Sonnet otherwise
```

**3단계 — Plan 모드로 전환해요**

`/plan`을 입력하거나, **Shift + Tab**을 두 번 눌러도 돼요.

```
> /plan
```

이렇게 하면 계획은 Opus가, 코드 작성은 Sonnet이 담당하는 최적의 조합이 만들어져요.

| 단계 | 사용 모델 | 역할 |
|------|-----------|------|
| `/plan` 모드 | **Opus** | 코드베이스 분석, 구조 파악, 계획 수립 |
| 코드 작성 | **Sonnet** | 빠르고 안정적인 구현 |

자세한 내용은 [`/model` 레퍼런스](/reference/commands/model)와 [`/plan` 레퍼런스](/reference/commands/plan)를 참고하세요.

## 이제 Claude Code에게 말을 걸어볼게요

Plan 모드에서 자연어로 원하는 것을 설명하면 돼요.\
완벽하게 정리된 기획서가 없어도 괜찮아요 — 아이디어를 편하게 말하면 Claude가 구체화해줘요.

아래는 포트폴리오 사이트를 시작할 때 쓸 수 있는 예시들이에요:

```
나만의 포트폴리오가 있는 웹사이트를 만들고 싶어.
```

```
개발자 포트폴리오 사이트 만들어줘.
자기소개, 프로젝트 목록, 연락처 섹션이 있었으면 좋겠어.
```

```
심플하고 깔끔한 포트폴리오 웹사이트를 만들고 싶어.
Next.js 쓰고 싶고, 다크 모드도 지원하면 좋겠어.
프로젝트 카드들이 그리드로 보이는 레이아웃이면 좋겠어.
```

```
포트폴리오 사이트 만들어줘.
히어로 섹션에 내 이름이랑 한 줄 소개가 나오고,
아래에 내가 만든 프로젝트들이 카드 형태로 보이는 구성이면 돼.
GitHub 링크도 넣을 수 있으면 좋겠어.
```
