# 03. 계획(PLAN)하기

## 원래는 이런 것들을 해야 해요

본격적인 개발 전에는 보통 이런 과정을 거쳐요:

- **브레인스토밍** — 어떤 제품을 만들지, 핵심 가치는 무엇인지
- **기능 정의** — 어떤 기능을 넣을지, 우선순위는 어떻게 할지
- **경쟁 제품 리서치** — 비슷한 제품들의 특징, 강점, 차별화 포인트 분석
- **사용자 흐름 설계** — 사용자가 어떤 경로로 움직이는지
- **기술 스택 결정** — 어떤 언어, 프레임워크, 인프라를 쓸지

<Callout type="info">
  실제 서비스를 만들 때는 이 과정이 매우 중요해요.\
  하지만 이번 실습은 **심플한 포트폴리오 사이트**를 만드는 것이 목표라 이 과정은 건너뛸게요.
</Callout>

---

## 대신 Claude Code의 Plan 모드를 써볼게요

본격적인 기획 없이도 **Claude Code의 [`/plan` 모드](/reference/commands/plan)**로 비슷한 효과를 낼 수 있어요.

Plan 모드에서 Claude는:
- 파일을 **절대 수정하지 않고** 분석만 해요
- 요구사항이 불명확하면 먼저 **질문을 던져요**
- 어떤 파일을 어떻게 수정할지 **단계별 계획**을 세워줘요
- 계획을 확인받은 후에만 실제 코드 작성을 시작해요

실수 없이 큰 그림을 먼저 잡을 수 있는 안전한 첫 단계예요.

---

## Opus Plan Mode를 쓰는 이유

[`/model`](/reference/commands/model)에서 **Opus Plan Mode**를 선택하면 계획 단계에서 Claude의 가장 강력한 모델인 **Opus**가 투입돼요.

| 단계 | 사용 모델 |
|------|-----------|
| 계획 (`/plan` 모드) | Opus — 복잡한 구조 파악, 깊은 분석 |
| 실제 코드 작성 | Sonnet — 빠르고 안정적인 구현 |

<Callout type="tip">
  Opus는 코드베이스 전체를 더 깊이 이해하고, 놓칠 수 있는 엣지 케이스나 구조적 문제를 미리 짚어줘요.\
  계획 단계에서만 Opus를 쓰고 구현은 Sonnet으로 넘기는 방식이라 비용도 효율적이에요.
</Callout>

설정 방법:

```
> /model
❯ 4. Opus Plan Mode  Use Opus in plan mode, Sonnet otherwise
```

---

## 이제 Claude Code에게 말을 걸어볼게요

Plan 모드에서 자연어로 원하는 것을 설명하면 돼요.\
완벽하게 정리된 기획서가 없어도 괜찮아요 — 아이디어를 편하게 말하면 Claude가 구체화해줘요.

아래는 포트폴리오 사이트를 시작할 때 쓸 수 있는 예시들이에요:

```
나만의 포트폴리오가 있는 웹사이트를 만들고 싶어.
```

```
개발자 포트폴리오 사이트 만들어줘.
자기소개, 프로젝트 목록, 연락처 섹션이 있었으면 좋겠어.
```

```
심플하고 깔끔한 포트폴리오 웹사이트를 만들고 싶어.
Next.js 쓰고 싶고, 다크 모드도 지원하면 좋겠어.
프로젝트 카드들이 그리드로 보이는 레이아웃이면 좋겠어.
```

```
포트폴리오 사이트 만들어줘.
히어로 섹션에 내 이름이랑 한 줄 소개가 나오고,
아래에 내가 만든 프로젝트들이 카드 형태로 보이는 구성이면 돼.
GitHub 링크도 넣을 수 있으면 좋겠어.
```

<Callout type="tip">
  처음엔 짧게 시작해도 돼요.\
  Claude가 질문을 던지면 그때 답하면서 구체화하는 것이 자연스러운 흐름이에요.
</Callout>
