- <Lightbulb size={16} color="#f59e0b" style={{display:'inline', verticalAlign:'middle', marginRight:'6px'}} />**브레인스토밍** — 어떤 제품을 만들지, 핵심 가치는 무엇인지

## 대신 클로드코드의 Plan 모드를 써볼게요

본격적인 기획 없이도 **클로드코드의 [`/plan` 모드](/reference/commands/plan)** 로 비슷한 효과를 낼 수 있어요.

Plan 모드에서 Claude는 파일을 **절대 수정하지 않아요**.\
코드를 읽고 분석하는 것만 할 수 있어요.\
요구사항이 불명확하면 먼저 질문을 던지고, 충분히 파악한 뒤 어떤 파일을 어떤 순서로 수정할지 단계별 계획을 만들어줘요.\
계획을 확인받은 후에만 실제 코드 작성을 시작하기 때문에 의도치 않은 수정이 일어나지 않아요.

아래가 실제로 Claude가 만들어주는 계획 출력이에요:

```
⏺ 알겠어요. 구현 계획을 정리했어요:

  ## 포트폴리오 사이트 계획

  ### 만들 파일
  - app/page.tsx            — 메인 페이지 (Hero + Projects + Contact)
  - app/layout.tsx          — 공통 레이아웃, 다크 모드 설정
  - components/Hero.tsx     — 이름, 한 줄 소개, CTA 버튼
  - components/Projects.tsx — 프로젝트 카드 그리드
  - components/Contact.tsx  — 이메일 연락처
  - data/projects.ts        — 프로젝트 데이터 (여기서 직접 수정)

  ### 작업 순서
  1. Next.js 프로젝트 생성 + Tailwind CSS, shadcn/ui 설치
  2. 다크 모드 + 모바일 반응형 레이아웃 설정
  3. Hero 섹션 구현
  4. 프로젝트 카드 그리드 구현
  5. 연락처 섹션 구현
  6. Vercel 배포 설정

  이 계획대로 진행할까요?
```

실수 없이 큰 그림을 먼저 잡을 수 있는 안전한 첫 단계예요.

## Plan 모드 전에 먼저 모델을 설정해요

Plan 모드를 시작하기 **전에** 모델을 **Opus Plan Mode**로 바꿔두는 게 좋아요.

<Callout type="info">
  <span className="inline-block text-xs font-semibold tracking-widest uppercase px-2 py-0.5 mb-2 bg-emerald-500/15 text-emerald-600 dark:text-emerald-400 border border-emerald-500/30">Best Practice</span>
  기본 모델(Sonnet)로도 Plan 모드를 쓸 수 있지만,\
  **Opus Plan Mode**로 설정하면 계획 단계에서 Claude의 가장 강력한 모델인 Opus가 투입돼 더 깊이 분석해줘요.\
  코드 작성 단계는 다시 Sonnet으로 자동 전환되니 비용도 효율적이에요.
</Callout>

**1단계 — 클로드코드 터미널에서 `/model`을 입력해요**

<VibeBlock>
```
> /model
```
</VibeBlock>

**2단계 — 메뉴에서 `Opus Plan Mode`를 선택해요**

방향키로 4번 항목으로 이동한 뒤 Enter를 눌러요.

```
  1. Default (recommended)  Opus 4.7 with 1M context
  2. Sonnet                 Sonnet 4.6
  3. Haiku                  Haiku 4.5
❯ 4. Opus Plan Mode ✔       Use Opus in plan mode, Sonnet otherwise
```

**3단계 — Plan 모드로 전환해요**

`/plan`을 입력하거나, **Shift + Tab**을 두 번 눌러도 돼요.

<VibeBlock>
```
> /plan
```
</VibeBlock>

이렇게 하면 계획은 Opus가, 코드 작성은 Sonnet이 담당하는 최적의 조합이 만들어져요.

| 단계 | 사용 모델 | 역할 |
|------|-----------|------|
| `/plan` 모드 | **Opus** | 코드베이스 분석, 구조 파악, 계획 수립 |
| 코드 작성 | **Sonnet** | 빠르고 안정적인 구현 |

자세한 내용은 [`/model` 레퍼런스](/reference/commands/model)와 [`/plan` 레퍼런스](/reference/commands/plan)를 참고하세요.

## 이제 클로드코드에게 말을 걸어볼게요

Plan 모드에서 자연어로 원하는 것을 설명하면 돼요.\
완벽하게 정리된 기획서가 없어도 괜찮아요 — 아이디어를 편하게 말하면 Claude가 구체화해줘요.

이번 실습에서 사용할 기술 스택은 아래와 같아요. Claude에게 말을 걸 때 이 스택을 함께 알려주면 더 정확하게 세팅해줘요.

<Callout type="tip">
  <span className="inline-block text-xs font-semibold tracking-widest uppercase px-2 py-0.5 mb-3 bg-emerald-500/15 text-emerald-600 dark:text-emerald-400 border border-emerald-500/30">이번 실습 기술 스택</span>
  
  | 기술 | 한 줄 설명 |
  |------|-----------|
  | **Next.js** | 웹사이트를 만드는 React 기반 프레임워크예요. 폴더 구조만으로 페이지가 자동으로 만들어져요. |
  | **TypeScript** | JavaScript에 타입을 추가한 언어예요. 코드 실수를 미리 잡아줘서 더 안전한 개발이 가능해요. |
  | **App Router** | Next.js의 최신 라우팅 방식이에요. `app/` 폴더 안에 파일을 만들면 자동으로 URL이 생성돼요. |
  | **Supabase** | 데이터베이스 + 로그인 기능을 쉽게 쓸 수 있는 서비스예요. 백엔드 없이도 데이터를 저장할 수 있어요. |
  | **Vercel** | Next.js를 만든 회사가 운영하는 배포 플랫폼이에요. GitHub에 올리면 자동으로 배포돼요. |
  | **Tailwind CSS** | 클래스 이름으로 디자인하는 CSS 도구예요. `text-lg`, `bg-blue-500` 같은 이름으로 스타일을 바로 적용해요. |
  | **shadcn/ui** | 버튼, 카드, 모달 같은 UI 컴포넌트 모음이에요. 복사해서 바로 쓸 수 있고 커스터마이징도 쉬워요. |
</Callout>

## 이런 식으로 대화가 흘러가요

Plan 모드를 시작하면 Claude가 바로 코드를 짜지 않아요.\
먼저 뭘 만들고 싶은지 물어보고, 충분히 파악한 뒤에 계획을 세워요.\
마치 개발자 친구와 아이디어를 정리하는 대화 같은 느낌이에요.

---

### 1단계 — 내가 무엇을 만들고 싶은지 말해요

<VibeBlock>
```
> 나만의 포트폴리오가 있는 웹사이트를 만들고 싶어.
  Next.js App Router, TypeScript, Tailwind CSS, shadcn/ui 기술스택을 사용할거고
  DB연동은 Supabase, 배포는 Vercel을 이용할 거야.
```
</VibeBlock>

---

### 2단계 — Claude가 구체적으로 물어봐요

Claude가 모호한 부분을 스스로 가정하지 않고 먼저 확인해요.

```
⏺ 포트폴리오 사이트 계획을 시작하기 전에 몇 가지 여쭤볼게요.

  1. 어떤 페이지나 섹션이 필요하신가요?
     (예: 자기소개 히어로, 프로젝트 목록, 기술 스택, 경력, 연락처 등)

  2. 프로젝트 데이터는 어떻게 관리하실 건가요?
     - 코드 파일에 직접 작성 (간단하고 빠름)
     - Supabase DB에서 불러오기 (나중에 관리자 페이지 추가 가능)

  3. 다크 모드가 필요하신가요?

  4. 모바일에서도 잘 보여야 하나요?
```

---

### 3단계 — 내가 답해요

`>` 뒤에 타이핑하고 Enter를 누르면 돼요.\
완벽하게 답하지 않아도 괜찮아요. 생각나는 대로 말하면 Claude가 맞춰줘요.

<VibeBlock>
```
> 섹션은 자기소개, 프로젝트 목록, 연락처면 충분해.
  프로젝트 데이터는 일단 코드에 직접 쓸게.
  다크 모드랑 모바일 대응 둘 다 필요해.
```
</VibeBlock>

모르거나 정해지지 않은 건 그냥 말해도 돼요:

<VibeBlock>
```
> DB 연동은 나중에 추가하고 싶은데 일단 없이 시작해도 될까?

⏺ 네, 전혀 문제없어요. 지금은 코드에 데이터를 직접 작성하고,
  나중에 Supabase 연동이 필요하면 그때 추가하는 방식으로 할게요.
```
</VibeBlock>

### 4단계 — Claude가 계획을 만들어줘요

대화가 끝나면 Claude가 어떤 파일을 만들고 어떤 순서로 작업할지 정리해줘요.

```
⏺ 알겠어요. 구현 계획을 정리했어요:

  ## 포트폴리오 사이트 계획

  ### 만들 화면
  - 메인 페이지 — Hero(자기소개), 프로젝트 목록, 연락처를 한 페이지에 배치

  ### 만들 파일
  - app/page.tsx            — 메인 페이지
  - app/layout.tsx          — 공통 레이아웃, 다크 모드 설정
  - components/Hero.tsx     — 이름, 한 줄 소개, 버튼
  - components/Projects.tsx — 프로젝트 카드 그리드
  - components/Contact.tsx  — 이메일 연락처
  - data/projects.ts        — 프로젝트 데이터 (여기서 직접 수정해요)

  ### 작업 순서
  1. Next.js 프로젝트 생성 + Tailwind, shadcn/ui 설치
  2. 다크 모드 + 모바일 반응형 레이아웃 설정
  3. Hero 섹션 구현
  4. 프로젝트 카드 그리드 구현
  5. 연락처 섹션 구현
```

```
  $ npm run dev
```
