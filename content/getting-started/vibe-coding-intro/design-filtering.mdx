# 10. 디자인 필터링

## 섹션과 컴포넌트 용어에 익숙해지세요

Claude에게 디자인을 맡기려면 **내가 원하는 걸 정확히 말할 수 있어야** 해요.\
그러려면 웹페이지를 구성하는 기본 단위의 이름을 알아야 해요.

용어를 모르면 "저기 위에 큰 부분"처럼 설명하게 되고, Claude는 엉뚱한 곳을 수정해요.\
반대로 이름을 알면 한 단어로 정확하게 전달할 수 있어요.

---

### 페이지 단위: 섹션(Section)

**섹션**은 페이지를 세로로 나누는 큰 구역이에요.\
스크롤하면 순서대로 지나치는 각각의 덩어리를 섹션이라고 불러요.

<SectionMap />

| 섹션 이름 | 역할 |
|-----------|------|
| **Navbar** | 상단 고정 내비게이션 바 |
| **Hero** | 페이지 최상단. 핵심 메시지와 CTA 버튼이 있어요 |
| **Features** | 서비스의 주요 기능을 나열하는 섹션 |
| **How it works** | 사용 방법이나 흐름을 설명하는 섹션 |
| **Pricing** | 요금제를 비교하는 섹션 |
| **Testimonials** | 사용자 후기나 인용문을 보여주는 섹션 |
| **FAQ** | 자주 묻는 질문과 답변 섹션 |
| **CTA** | 행동 유도 섹션 — "지금 시작하기" 같은 버튼이 크게 있어요 |
| **Footer** | 페이지 맨 아래. 링크, 저작권 정보 등 |

<Callout type="tip">
"Hero 섹션 배경색 바꿔줘", "Pricing 섹션 카드 3개로 만들어줘"처럼 섹션 이름을 쓰면 Claude가 정확히 찾아가요.
</Callout>

---

### 재사용 단위: 컴포넌트(Component)

**컴포넌트**는 섹션 안을 채우는 작은 UI 조각이에요.\
버튼 하나, 카드 하나, 입력창 하나가 각각 컴포넌트예요.

<ShadcnComponentGrid />

<Callout type="tip">
"Features 섹션 안에 Card 컴포넌트 4개 넣어줘"처럼 섹션 + 컴포넌트를 함께 쓰면 훨씬 정확하게 동작해요.
</Callout>
