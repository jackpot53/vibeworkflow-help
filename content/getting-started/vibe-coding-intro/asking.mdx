# 09. 물어보는 것도 실력이다

바이브코딩에서 코드를 짜는 건 Claude가 해줘요.\
하지만 **무엇을 만들지 설명하는 건 나의 몫**이에요.

같은 기능을 원해도 어떻게 말하느냐에 따라 결과가 완전히 달라져요.\
좋은 질문은 좋은 코드를 만들고, 모호한 질문은 엉뚱한 결과를 만들어요.

---

## 구체적일수록 좋아요

Claude는 맥락이 많을수록 정확하게 동작해요.\
"만들어줘" 한 마디보다 목적, 조건, 제약을 함께 말해주면 훨씬 나은 결과가 나와요.

| 모호한 요청 | 구체적인 요청 |
|-------------|--------------|
| 버튼 만들어줘 | 로그인 버튼 만들어줘. 클릭하면 `/api/auth` 로 POST 요청 보내고, 로딩 중엔 비활성화돼야 해. |
| 에러 고쳐줘 | `TypeError: Cannot read properties of undefined` 이 에러 고쳐줘. `user.name`을 읽으려는데 `user`가 null일 때 발생해. |
| 리스트 보여줘 | 프로젝트 목록을 카드 형태로 보여줘. 카드마다 제목, 설명, GitHub 링크가 있어야 해. |
| 디자인 바꿔줘 | Hero 섹션 배경을 짙은 남색으로 바꾸고, 텍스트는 흰색으로 해줘. 다크 모드에서도 같은 색이면 돼. |

---

## 모르면 모른다고 말해도 괜찮아요

기술 용어를 몰라도, 정확한 스펙을 몰라도 괜찮아요.\
느낌이나 원하는 결과를 말하면 Claude가 구체화해줘요.

<VibeBlock>
```

```
</VibeBlock>

<VibeBlock>
```
> 지금 너무 밋밋해. 좀 더 세련된 느낌으로 바꿔줘.
```
</VibeBlock>

<VibeBlock>

```
> 사용자가 실수로 삭제할까봐 걱정돼. 확인 창 같은 걸 넣어줄 수 있어?
```
</VibeBlock>

Claude가 의도를 파악하고 알아서 구현해줘요.\
확실하지 않을 땐 먼저 물어봐줘요.

---

## 한 번에 다 요청하지 않아도 돼요

복잡한 기능을 한 번에 설명하려다 보면 오히려 요청이 길어지고 결과가 산으로 가요.\
**작게 나눠서 하나씩** 요청하는 게 훨씬 깔끔해요.

## 수정할 위치를 정확히 알려줘요 — devlabel

컴포넌트가 많아지면 "그 파란 버튼 있는 카드 수정해줘"처럼 설명이 모호해지기 쉬워요.\
AI는 파일과 컴포넌트 이름을 보고 위치를 추론하는데, 비슷한 이름이 여러 개면 헷갈려요.

`Header`, `MainHeader`, `PageHeader`, `SiteHeader`가 다 있을 때 `"Header 수정해줘"`라고 하면 AI가 어디를 고쳐야 할지 몰라요.

**devlabel**을 붙이면 이 문제가 해결돼요.\
각 영역 모서리에 컴포넌트 이름이 작게 표시되고, 클릭하면 `파일명 - 컴포넌트명` 형태로 클립보드에 복사돼요.\
로컬 개발 환경에서만 보이고 실제 배포 사이트에선 자동으로 사라져요.

아래 프롬프트를 클로드코드에 한 번만 실행해두면 돼요.

<Callout type="tip">
  <span className="inline-block text-xs font-semibold tracking-widest uppercase px-2 py-0.5 mb-2 bg-blue-500/15 text-blue-600 dark:text-blue-400 border border-blue-500/30">실습</span>
  아래 프롬프트를 그대로 클로드코드에 입력해보세요.
</Callout>

<VibeBlock>
```
페이지에 리액트 컴포넌트명이 명시적으로 보이도록 DevLabel 컴포넌트를 만들고 붙여줘.

**DevLabel 스펙:**
- depth prop으로 섹션(1) / 컴포넌트(2) / 하위 영역(3) 세 계층을 구분해줘
- 계층별로 라벨 위치와 색이 달라야 해:
    - depth=1 (섹션): 파란색, top-left 코너
    - depth=2 (컴포넌트): 보라색, top-right 코너
    - depth=3 (하위 영역): 주황색, bottom-left 코너
- 클릭 시 클립보드에 "파일명 - 컴포넌트명" 형태로 복사돼야 해
  예) components/ProjectCard.tsx - ProjectCard
- NODE_ENV가 development일 때만 렌더링되고, production에서는 완전히 사라져야 해

**적용 범위:**
- 지금 있는 모든 컴포넌트에 적절한 depth로 DevLabel을 붙여줘
- 앞으로 컴포넌트를 새로 만들 때마다 DevLabel을 자동으로 붙이도록 CLAUDE.md에 규칙으로 추가해줘
```
</VibeBlock>
